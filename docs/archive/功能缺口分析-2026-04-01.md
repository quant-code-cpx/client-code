## 二、功能缺口一：首页 Dashboard 仍为模板占位

### 2.1 现状

当前首页 `src/sections/overview/` 渲染的是 Minimal UI 模板自带的通用分析卡片：

- "Weekly Sales" / "New Users" / "Purchase Orders" / "Messages" 四个统计卡片
- "Website Visits" / "Current Visits" 等与量化无关的图表
- 新闻摘要、订单时间线等电商模板组件
- 全部使用 `src/_mock/` 模拟数据，未接入真实 API

### 2.2 应有的量化首页

| 组件 | 说明 | 数据来源 |
|------|------|----------|
| **市场快照卡片** | 沪深 300 / 中证 500 / 上证 / 深证 实时指数 + 涨跌 | `market.fetchIndexQuote()` |
| **市场情绪仪表盘** | 涨跌家数比 / 涨停板数 / 跌停板数 / 红绿比例条 | `market.fetchSentiment()` |
| **自选股动态** | 用户关注标的今日涨跌排列（需后端自选股 API） | 待开发 |
| **策略预警** | 今日触发的选股条件/价格预警汇总 | 待开发 |
| **最近回测** | 最近 5 条回测任务的状态和核心指标 | `backtest.getRuns()` |
| **资金流向概览** | 主力 / 散户 / 北向资金当日净流向 | `market.fetchMoneyFlow()` |
| **热门板块** | 今日涨幅前 5 / 资金净流入前 5 板块 | `market.fetchSectorRanking()` |
| **系统状态** | 数据同步状态 / 最后更新时间（仅管理员可见） | `tushareSync.getPlans()` |

### 2.3 前端工作量估算

- 替换 overview section：~3-5 天
- 需新增 API：自选股动态、策略预警（依赖后端开发）
- 可复用现有市场 API 的数据

## 三、功能缺口二：缺少自选股与研究工作台

### 3.1 问题

后端痛点文档明确指出：系统是"工具型"——用户查完即走，无法沉淀研究过程。当前前端：

- 无自选股管理页面
- 无研究笔记功能
- 无条件订阅 / 定时推送
- 无策略草稿箱
- 股票详情页无"加入自选"按钮

### 3.2 需新增页面

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| **自选股管理** | `/watchlist` | 创建/编辑自选组、添加/移除标的、批量导入、标签管理 |
| **自选股详情** | `/watchlist/:id` | 组内标的涨跌排列、K 线缩略图、分组笔记 |
| **研究笔记** | `/notes` | 按股票/标签/时间浏览研究记录，Markdown 编辑器 |
| **条件订阅** | `/subscriptions` | 已保存的筛选条件列表、运行频率、最近推送结果 |
| **策略草稿** | `/backtest/drafts` | 未提交的回测配置暂存列表，一键加载到工作台 |

### 3.3 页面间联动

- 股票详情页头部添加 ⭐ "加入自选" 按钮
- 选股器结果添加 "全部加入自选" 操作
- 回测工作台添加 "保存为草稿" 按钮
- 因子选股结果添加 "导入自选" 操作
- 首页 Dashboard 展示自选股动态卡片

### 3.4 依赖后端 API

```
CRUD /api/watchlist
CRUD /api/watchlist/:id/items
CRUD /api/research-notes
CRUD /api/screener-subscriptions
CRUD /api/strategy-drafts
```

## 四、功能缺口三：缺少数据导出能力

### 4.1 问题

当前所有数据仅在页面渲染，用户无法：

- 导出股票列表 / 选股结果为 CSV / Excel
- 导出回测报告为 PDF
- 导出因子截面数据
- 导出 K 线 / 财务数据供 Python 分析
- 分享回测结果给其他用户

### 4.2 需新增功能

| 功能 | 位置 | 实现方式 |
|------|------|----------|
| **表格导出 CSV** | 股票列表、选股结果、因子截面、回测交易记录 | 前端使用 `papaparse` 或原生 Blob 生成 CSV |
| **表格导出 Excel** | 同上 | 前端使用 `xlsx` 库或后端生成 |
| **回测报告 PDF** | 回测详情页 | 后端生成 PDF（含图表），前端下载 |
| **图表导出图片** | 所有 ApexCharts 图表 | ApexCharts 内置 `exportToSVG()` / `exportToPNG()` |
| **分享链接** | 回测详情页 | 后端生成只读分享 Token，前端展示分享弹窗 |

### 4.3 实现建议

- **表格导出**：可纯前端实现（无需等后端），在 Table Toolbar 添加导出按钮
- **图表导出**：ApexCharts 已支持 `toolbar.export`，只需在 `useChart` 中启用
- **PDF 报告**：需后端配合（推荐 Puppeteer 或 @nestjs/bull + headless Chrome）

## 五、功能缺口四：实时行情与预警页面空缺

### 5.1 现状

- WebSocket 基础设施已就绪（Socket.IO，已用于回测进度 + 同步通知）
- **无任何盘中实时行情展示**
- **无价格预警配置页面**
- **无异动监控页面**

### 5.2 需新增页面

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| **实时行情面板** | `/market/realtime` | 按自选股/板块订阅实时报价，5 秒级刷新，闪烁高亮涨跌 |
| **价格预警管理** | `/alerts` | 创建/编辑/删除预警规则（股价突破/跌破/成交量放大） |
| **预警历史** | `/alerts/history` | 已触发预警的时间线记录 |
| **异动监控** | `/market/anomaly` | 连板监测、大单成交、成交量异常放大的实时推送列表 |
| **事件日历** | `/calendar` | 财报发布日、解禁日、分红除权日的日历视图 |

### 5.3 WebSocket 扩展

当前 `src/lib/socket.ts` 需扩展以下事件：

```typescript
// 实时行情
socket.on('realtime_quote', (data: RealtimeQuote) => { ... })

// 价格预警
socket.on('price_alert_triggered', (data: AlertEvent) => { ... })

// 异动通知
socket.on('anomaly_detected', (data: AnomalyEvent) => { ... })
```

### 5.4 依赖后端 API

- 盘中实时行情数据源（Tushare 实时接口或第三方 WebSocket 源）
- 预警规则 CRUD API
- 异动检测引擎
- 事件日历数据聚合 API

## 九、功能缺口八：数据同步管理台需升级

### 9.1 现状

当前 `tushare-sync-view.tsx` 是一个 469 行的单文件组件，功能包括：
- 同步计划列表（按分类分组）
- 增量/全量模式切换
- 手动触发同步
- WebSocket 进度监听

### 9.2 问题

后端文档建议的"数据同步可视化管理台"包括更丰富的能力：

| 缺失功能 | 说明 |
|----------|------|
| **数据表同步状态总览** | 各表最后同步时间、数据行数、缺失天数 — 当前仅有计划列表，无状态统计 |
| **同步任务甘特图** | 各任务耗时、依赖关系 — 当前无可视化 |
| **数据完整性检查** | 哪些股票哪些日期缺数据 — 当前无此能力 |
| **异常数据标记** | 人工修正入口 — 当前无此能力 |
| **同步历史日志** | 历史同步记录（成功/失败/耗时） — 当前仅有实时通知 |

### 9.3 重构建议

将单文件拆分为组件化结构：

```
src/sections/tushare-sync/
├── view/
│   └── tushare-sync-view.tsx           ← 主视图
├── sync-plan-table.tsx                 ← 同步计划表格
├── sync-status-overview.tsx            ← 状态统计卡片
├── sync-progress-tracker.tsx           ← 进度追踪
├── sync-history-log.tsx                ← 历史日志表格
├── sync-data-integrity-panel.tsx       ← 数据完整性面板
├── sync-gantt-chart.tsx                ← 甘特图
├── hooks/
│   └── use-sync-notification.ts        ← 从 context 提取
└── types.ts
```

## 十三、优化点二：数据获取与状态管理

### 13.1 当前问题

- 每个页面使用 `useState` + `useEffect` + `fetch` 手动管理请求生命周期
- 无请求缓存 — 切换页面再切回来会重新请求
- 无请求去重 — 同时渲染两个需要相同数据的组件会发两次请求
- 无乐观更新 — 所有修改操作需等服务端响应后才刷新 UI
- 跨组件状态共享困难 — 如选股条件、回测参数等

### 13.2 建议方案

| 改进 | 方案 | 影响范围 |
|------|------|----------|
| **数据请求库** | 引入 TanStack Query（React Query） | 全局 |
| **好处** | 自动缓存、去重、重试、后台刷新、SSR 就绪 | — |
| **跨页面状态** | 引入 Zustand（轻量级状态管理） | 筛选条件、用户偏好、回测参数 |
| **请求取消** | 已支持 AbortSignal（API Client 已实现），但页面未使用 | 长列表 / 搜索页 |

### 13.3 迁移路径

1. 安装 `@tanstack/react-query`
2. 在 `app.tsx` 添加 `QueryClientProvider`
3. 逐页面将 `useEffect` + `useState` 模式迁移为 `useQuery` / `useMutation`
4. 优先迁移高频访问页面（股票列表、市场概览）

## 十四、优化点三：图表交互体验增强

### 14.1 当前问题

- K 线图缺少十字光标联动
- 图表缩放仅支持 ApexCharts 默认行为，无自定义区间选择
- 技术指标叠加需要切换 Tab，无法多指标同时显示
- 图表无导出功能（PNG / SVG）
- 大数据量图表（如全市场散点图）无降采样处理

### 14.2 建议改进

| 改进 | 说明 | 优先级 |
|------|------|--------|
| **十字光标** | K 线图添加十字光标，悬浮显示 OHLC + 成交量 + 指标值 | 高 |
| **区间选择** | 双指/鼠标框选放大，带迷你导航图 | 中 |
| **多指标叠加** | 允许同时显示 MA + MACD + 成交量在同一图表 | 高 |
| **图表导出** | 启用 ApexCharts toolbar 的 export 功能 | 低 |
| **数据降采样** | 超过 500 点时使用 LTTB 算法降采样 | 中 |
| **图表联动** | 同一页面多个图表的 X 轴（时间）联动滚动 | 中 |

## 十五、优化点四：性能优化

### 15.1 当前瓶颈

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| **长列表无虚拟滚动** | 股票列表 5000+ 条渲染卡顿 | 引入 `@tanstack/react-virtual` 或 `react-window` |
| **无 Bundle 分析** | 无法发现体积异常的依赖 | 添加 `rollup-plugin-visualizer` |
| **未使用 React.memo** | 高频重渲染的表格行/图表组件 | 识别并 memo 化热点组件 |
| **图表渲染阻塞** | 多图表同时渲染导致页面卡顿 | 使用 `IntersectionObserver` 延迟渲染可视区域外的图表 |
| **无路由预加载** | 导航跳转时白屏等待 | `<Link>` 组件 hover 时预取下一页 chunk |
| **图片未优化** | 静态资源无 CDN / 压缩 | Vite build 配置优化，Vercel 自动 CDN |
