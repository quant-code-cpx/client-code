## 二、完全缺失的模块（0% 前端覆盖）

### 🔴 Portfolio — 组合管理（20 端点）

| 后端端点                         | 说明                   |
| -------------------------------- | ---------------------- |
| `POST /portfolio/create`         | 创建组合               |
| `POST /portfolio/list`           | 获取我的组合列表       |
| `POST /portfolio/detail`         | 获取组合详情（含持仓估值） |
| `POST /portfolio/update`         | 更新组合基本信息       |
| `POST /portfolio/delete`         | 删除组合               |
| `POST /portfolio/holding/add`    | 添加持仓（或加仓）     |
| `POST /portfolio/holding/update` | 更新持仓数量和成本     |
| `POST /portfolio/holding/remove` | 删除持仓               |
| `POST /portfolio/pnl/today`      | 当日浮动盈亏           |
| `POST /portfolio/pnl/history`    | 历史净值曲线           |
| `POST /portfolio/risk/industry`  | 行业分布分析           |
| `POST /portfolio/risk/position`  | 仓位集中度分析         |
| `POST /portfolio/risk/market-cap`| 市值分布分析           |
| `POST /portfolio/risk/beta`      | Beta 系数分析          |
| `POST /portfolio/rule/list`      | 获取风控规则列表       |
| `POST /portfolio/rule/upsert`    | 创建或更新风控规则     |
| `POST /portfolio/rule/update`    | 修改风控规则           |
| `POST /portfolio/rule/delete`    | 删除风控规则           |
| `POST /portfolio/risk/check`     | 执行风控规则检测       |
| `POST /portfolio/risk/violations`| 查询历史违规记录       |

**建议**：组合管理是量化投资闭环的核心，含组合 CRUD、持仓管理、盈亏追踪、风险分析和风控检测五大功能块，优先级 **高**。

### 🔴 Event Study — 事件驱动研究（9 端点）

| 后端端点                              | 说明                                  |
| ------------------------------------- | ------------------------------------- |
| `GET /event-study/event-types`        | 获取系统支持的事件类型列表            |
| `POST /event-study/events`            | 分页查询指定类型的事件记录            |
| `POST /event-study/analyze`           | 事件影响分析（计算超额收益 AAR/CAAR） |
| `POST /event-study/signal-rules`      | 创建事件信号规则                      |
| `POST /event-study/signal-rules/list` | 查询我的信号规则列表                  |
| `PATCH /event-study/signal-rules/:id` | 更新事件信号规则                      |
| `DELETE /event-study/signal-rules/:id`| 删除事件信号规则（软删除）            |
| `POST /event-study/signal-rules/scan` | 手动触发事件信号扫描（管理员）        |
| `POST /event-study/signals`           | 查询已触发的事件信号历史              |

**建议**：事件驱动研究可提升量化研究深度，含事件分析和信号规则两大功能模块，优先级 **高**。

### 🔴 Alert — 预警与监控（8 端点）

| 后端端点                        | 说明                                                 |
| ------------------------------- | ---------------------------------------------------- |
| `GET /alert/calendar`           | 查询事件日历（财报披露/限售解禁/除权除息/业绩预告）  |
| `POST /alert/price-rules`       | 创建价格预警规则                                     |
| `GET /alert/price-rules`        | 查询我的价格预警规则列表                             |
| `PATCH /alert/price-rules/:id`  | 更新价格预警规则                                     |
| `DELETE /alert/price-rules/:id` | 删除价格预警规则（软删除）                           |
| `POST /alert/price-rules/scan`  | 手动触发价格预警扫描（管理员）                       |
| `GET /alert/anomalies`          | 查询异动监控记录                                     |
| `POST /alert/anomalies/scan`    | 手动触发异动监控扫描（管理员）                       |

**建议**：预警监控是投资者日常使用频率很高的功能，含事件日历、价格预警和异动监控三大功能，优先级 **高**。

### 🔶 Report — 量化报告（6 端点）

| 后端端点                 | 说明             |

| `POST /report/backtest`  | 生成回测报告     |

**建议**：量化报告功能可为回测、个股、组合自动生成分析报告，提升研究输出效率，优先级 **中**。

### 🔶 Heatmap — 市场热力图（3 端点）

| 后端端点                         | 说明                                     |
| -------------------------------- | ---------------------------------------- |
| `POST /heatmap/data`             | 获取市场热力图数据（涨跌幅分布）         |
| `POST /heatmap/snapshot/trigger` | 手动触发热力图快照聚合（管理员）         |

**建议**：可嵌入市场概览页，作为行业/个股涨跌分布可视化补充，优先级 **中**。

### 🔶 Index — 指数模块（3 端点）

| 后端端点                   | 说明               |

| `POST /index/list`         | 支持的核心指数列表 |
| `POST /index/daily`        | 指数日线行情       |
| `POST /index/constituents` | 指数成分股及权重   |

**建议**：Market 模块已通过 `/market/index-quote` + `/market/index-trend` 展示指数概览，但 Index 模块的日线详情和成分股权重尚未接入，可嵌入市场概览或独立详情页，优先级 **中**。

### 🔶 Pattern — 相似 K 线形态匹配（3 端点）

| 后端端点                         | 说明                                             |
| -------------------------------- | ------------------------------------------------ |

| `POST /pattern/search`           | 相似 K 线形态搜索（基于股票日线区间）             |
| `POST /pattern/search-by-series` | 相似形态搜索（基于自定义价格序列）                |

**建议**：K 线形态匹配是技术分析的增强功能，可嵌入股票详情页分析 Tab 或独立页面，优先级 **中**。

### Factor — 因子（前端 9/21，覆盖 43%）

#### ❌ 未接入的端点

| 后端端点                               | 说明                      |
| -------------------------------------- | ------------------------- |
| `POST /factor/analysis/orthogonalize`  | 因子正交化                |
| `POST /factor/analysis/fama-macbeth`   | Fama-MacBeth 截面回归检验 |
| `POST /factor/custom/create`           | 创建自定义因子            |
| `POST /factor/custom/test`             | 试算表达式                |
| `POST /factor/custom/update`           | 更新自定义因子            |
| `POST /factor/custom/delete`           | 删除自定义因子            |
| `POST /factor/custom/precompute`       | 触发单因子预计算          |
| `POST /factor/admin/precompute`        | 手动触发因子预计算        |
| `POST /factor/admin/backfill`          | 触发历史因子值回补        |
| `POST /factor/admin/precompute/status` | 查询预计算状态            |
| `POST /factor/backtest/submit`         | 因子策略一键回测          |
| `POST /factor/backtest/attribution`    | 因子归因分析              |

**建议**：

- 自定义因子引擎（create/test/update/delete/precompute）是因子研究闭环的关键，优先级 **高**。
- 因子一键回测和归因分析与 Strategy/Backtest 联动，优先级 **高**。
- Fama-MacBeth 检验和正交化分析属于高级功能，优先级 **中**。
- Admin 接口（precompute/backfill/status）可纳入 Tushare 管理面板，优先级 **低**。
