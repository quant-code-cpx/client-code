# 市场热力图（Heatmap）— 前端实现规划

> **目标读者**：AI 代码生成助手。请严格按照本文定义的组件结构、布局规则、API 调用方式实现。

---

## 一、技术栈与约定

| 技术           | 版本/说明                                                   |
| -------------- | ----------------------------------------------------------- |
| 框架           | React 19（Vite）                                            |
| UI 库          | MUI v7（`@mui/material`），使用 `sx` prop 做样式            |
| 图表           | ApexCharts（`react-apexcharts` / `apexcharts`）             |
| API 客户端     | 项目内 `src/api/client.ts` 的 `apiClient.post()`            |
| 布局容器       | `DashboardContent`（来自 `src/layouts/dashboard`）          |
| 自定义图表封装 | `src/components/chart/` 下的 `Chart` 组件和 `useChart` hook |

### 关键约定

1. **API 调用**：统一使用 `apiClient.post<T>(url, body)` 方式发请求，不要用 fetch/axios。
2. **日期选择**：提供一个 `trade_date` 可选输入（YYYYMMDD 格式），不传则后端自动取最新交易日。
3. **错误处理**：组件内使用 `try/catch`，失败时在 UI 上显示提示，不能阻塞整页渲染。
4. **加载状态**：每个卡片组件独立管理 `loading` 状态，使用 MUI `Skeleton` 占位。
5. **图表 tooltip**：不要同时设置 `tooltip.shared = true` 和 `tooltip.intersect = true`，这会导致 ApexCharts 报错。如果使用 shared tooltip，必须设置 `tooltip.intersect = false`。
6. **空数据保护**：使用数据前必须检查是否为 null / undefined / 空数组，防止 `Cannot read properties of null` 错误。
7. **颜色规则**：涨 → 红色（`error.main`），跌 → 绿色（`success.main`），平 → 灰色（`text.secondary`）。
8. **金额单位**：Tushare `amount` 字段单位为千元，展示时需转换为亿元（÷100000）。

## 二、需求来源 & 后端端点

市场热力图模块对应 3 个后端端点：

| 后端端点                         | 说明                                    |
| -------------------------------- | --------------------------------------- |
| `POST /heatmap/data`             | 获取市场热力图数据（涨跌幅分布）        |
| `POST /heatmap/snapshot/trigger` | 手动触发热力图快照聚合（管理员）        |
| `POST /heatmap/snapshot/history` | 查询指定日期热力图快照（缓存/实时降级） |

### 功能定位

市场热力图是**市场概览的增强可视化**，以 TreeMap 形式直观展示全市场/行业维度的个股涨跌分布。核心场景：

1. **实时热力图**：当日全市场个股涨跌幅分布（按行业分组，区块大小 = 市值/成交额，颜色 = 涨跌幅）
2. **快照回放**：查看历史某日的热力图快照，支持日期切换
3. **管理员维护**：手动触发快照聚合（仅管理员可用）

## 六、API 层定义

在 `src/api/market.ts` 中新增以下类型和函数：

### 6.1 类型定义

```typescript
// ---------- Heatmap 热力图 ----------

/** 热力图单个股票数据 */
export type HeatmapStockItem = {
  /** 股票代码 */
  tsCode: string;
  /** 股票名称 */
  name: string;
  /** 所属行业 */
  industry: string;
  /** 涨跌幅 % */
  pctChg: number;
  /** 总市值（元） */
  totalMv: number;
  /** 流通市值（元） */
  circMv: number;
  /** 成交额（千元） */
  amount: number;
  /** 收盘价 */
  close: number;
};

/** 热力图行业汇总 */
export type HeatmapSectorSummary = {
  /** 行业名称 */
  industry: string;
  /** 行业涨跌幅 % */
  pctChg: number;
  /** 上涨家数 */
  upCount: number;
  /** 下跌家数 */
  downCount: number;
  /** 平盘家数 */
  flatCount: number;
  /** 行业总成交额（千元） */
  totalAmount: number;
};

/** POST /heatmap/data 响应 */
export type HeatmapDataResult = {
  /** 交易日期 YYYYMMDD */
  tradeDate: string;
  /** 个股列表 */
  stocks: HeatmapStockItem[];
  /** 行业汇总 */
  sectors: HeatmapSectorSummary[];
  /** 涨跌分布统计 */
  distribution: {
    /** 涨停家数 */
    limitUp: number;
    /** 跌停家数 */
    limitDown: number;
    /** 上涨家数 */
    upCount: number;
    /** 下跌家数 */
    downCount: number;
    /** 平盘家数 */
    flatCount: number;
    /** 各涨跌幅区间分布 [{range: "-10~-9", count: 5}, ...] */
    ranges: Array<{ range: string; count: number }>;
  };
};

/** POST /heatmap/snapshot/trigger 请求体 */
export type HeatmapSnapshotTriggerQuery = {
  trade_date?: string;
};

/** POST /heatmap/snapshot/trigger 响应 */
export type HeatmapSnapshotTriggerResult = {
  success: boolean;
  message: string;
  tradeDate: string;
};

/** POST /heatmap/snapshot/history 请求体 */
export type HeatmapSnapshotHistoryQuery = {
  trade_date: string;
};

/** POST /heatmap/snapshot/history 响应 — 与 HeatmapDataResult 结构相同（快照数据） */
export type HeatmapSnapshotHistoryResult = HeatmapDataResult & {
  /** 快照生成时间 ISO */
  snapshotTime: string;
  /** 是否来源于快照缓存（false 表示实时降级） */
  fromCache: boolean;
};
```

### 7.2 `HeatmapTreemapChart` — 核心 TreeMap 热力图

**文件**：`src/sections/market-heatmap/heatmap-treemap-chart.tsx`

**API**：`POST /api/heatmap/data`
**请求体**：`{ trade_date?: string }`

**Props**：

```typescript
type Props = {
  tradeDate?: string;
  groupBy: 'industry' | 'market'; // industry=按行业分组 market=不分组/按交易所
  sizeBy: 'totalMv' | 'circMv' | 'amount'; // 区块大小依据
};
```

**数据流转**：

1. 调用 `fetchHeatmapData({ trade_date: tradeDate })` 获取 `HeatmapDataResult`
2. 将 `stocks[]` 按 `groupBy` 分组，转换为 ApexCharts TreeMap 数据格式
3. 每个区块的 `x`（标签）= 股票名称，`y`（大小）= `sizeBy` 对应字段值
4. 颜色由 `pctChg` 映射

**ApexCharts TreeMap 数据格式**：

```typescript
// groupBy === 'industry' 时，使用多层级 treemap
const series = sectors.map((sector) => ({
  name: sector, // 行业名称作为分组标签
  data: stocksInSector.map((s) => ({
    x: s.name, // 股票名称
    y: s[sizeBy], // 区块大小
  })),
}));

// 注意：ApexCharts treemap 使用 series[].data[].x/y 格式
```

**颜色映射规则**（通过 `plotOptions.treemap.colorScale`）：

```typescript
plotOptions: {
  treemap: {
    distributed: false,  // 不要 distributed，使用 colorScale
    enableShades: false,
    colorScale: {
      ranges: [
        { from: -100, to: -7,   color: '#00796B', name: '跌≥7%' },    // 深绿
        { from: -7,   to: -3,   color: '#4CAF50', name: '跌3~7%' },   // 绿
        { from: -3,   to: -0.5, color: '#A5D6A7', name: '跌0.5~3%' }, // 浅绿
        { from: -0.5, to: 0.5,  color: '#9E9E9E', name: '平盘' },     // 灰
        { from: 0.5,  to: 3,    color: '#EF9A9A', name: '涨0.5~3%' }, // 浅红
        { from: 3,    to: 7,    color: '#F44336', name: '涨3~7%' },   // 红
        { from: 7,    to: 100,  color: '#B71C1C', name: '涨≥7%' },    // 深红
      ],
    },
  },
},
```

**⚠️ 关键：颜色数据传递**

ApexCharts TreeMap 的 colorScale 是根据 `y` 值着色的。但我们需要 `y` 表示市值（区块大小），颜色按 `pctChg`（涨跌幅）。解决方案：

- **方案 A（推荐）**：使用 `y` 传递涨跌幅用于着色，将实际大小值放入自定义字段（如 `meta`），通过 `tooltip.custom` 回调读取完整数据
- **方案 B**：使用 `fillColor` 逐个数据点设置颜色

选择**方案 B**，为每个数据点计算颜色：

```typescript
function getHeatmapColor(pctChg: number): string {
  if (pctChg <= -7) return '#00796B';
  if (pctChg <= -3) return '#4CAF50';
  if (pctChg <= -0.5) return '#A5D6A7';
  if (pctChg < 0.5) return '#9E9E9E';
  if (pctChg < 3) return '#EF9A9A';
  if (pctChg < 7) return '#F44336';
  return '#B71C1C';
}

// 数据点格式：
data: stocksInSector.map((s) => ({
  x: s.name,
  y: s[sizeBy],
  fillColor: getHeatmapColor(s.pctChg),
})),
```

**Tooltip 自定义**：

```typescript
tooltip: {
  custom({ seriesIndex, dataPointIndex, w }: any) {
    const data = w.config.series[seriesIndex].data[dataPointIndex];
    // data 中包含 x(名称)、y(市值/成交额)、fillColor
    // 需要从原始数据中找到对应股票完整信息
    // 建议：将完整 stock 对象存入组件 ref/state，通过 name 查找
    return `<div class="apexcharts-tooltip-custom" style="padding:8px">
      <b>${data.x}</b><br/>
      涨跌幅: ${pctChg.toFixed(2)}%<br/>
      收盘价: ${close.toFixed(2)}<br/>
      成交额: ${(amount / 100000).toFixed(2)}亿
    </div>`;
  },
},
```

**注意事项**：

- ⚠️ TreeMap 的 `type` 为 `'treemap'`
- ⚠️ 股票数量可能达到 4000+，大数据量下需注意性能。建议按市值/成交额排序后取 **Top 200~500** 展示，或按行业各取 Top N
- ⚠️ `chart.animations.enabled` 建议设为 `false` 以提升大数据量渲染性能
- ⚠️ 图表高度建议 `sx={{ height: 560 }}`，给足空间让标签可读
- ⚠️ `dataLabels.enabled: true`，显示股票名称；可设置 `minFontSize: 8, maxFontSize: 14` 根据区块大小自适应

**图表配置示例**：

```typescript
const chartOptions = useChart({
  chart: {
    type: 'treemap',
    toolbar: { show: false },
    animations: { enabled: false }, // 大数据量关闭动画
  },
  plotOptions: {
    treemap: {
      distributed: true, // 使用 fillColor 逐点着色时需要
      enableShades: false,
    },
  },
  dataLabels: {
    enabled: true,
    style: {
      fontSize: '11px',
      fontWeight: 500,
    },
    formatter(text: string, op: any) {
      // 显示：名称 + 涨跌幅
      const val = op.value;
      return [text, `${val >= 0 ? '+' : ''}${val.toFixed(1)}%`];
    },
    offsetY: -2,
  },
  tooltip: {
    // 使用 custom tooltip（见上）
  },
  legend: { show: false }, // TreeMap 不需要图例
});
```

**数据裁剪策略**：

```typescript
// 按 sizeBy 字段降序排序，取前 300 只
const topStocks = [...stocks].sort((a, b) => b[sizeBy] - a[sizeBy]).slice(0, 300);
```

### 7.5 `HeatmapSnapshotPanel` — 快照管理面板（管理员）

**文件**：`src/sections/market-heatmap/heatmap-snapshot-panel.tsx`

**权限控制**：

- 从 auth context 获取当前用户角色
- 非管理员返回 `null`（不渲染）

**API**：

- `POST /api/heatmap/snapshot/trigger`：触发快照
- `POST /api/heatmap/snapshot/history`：查询快照

**Props**：无（内部自行管理状态）

**UI 结构**：

- `Card` 包裹
- 卡片标题：`"快照管理"`，标题旁显示 `Label` 标签 `"管理员"`
- 上方操作栏：
  - 日期输入框（查询/触发的目标日期）
  - `Button`：`"触发快照聚合"`（确认弹窗后调用 `triggerHeatmapSnapshot`）
  - `Button`：`"查询快照"`（调用 `fetchHeatmapSnapshotHistory`）
- 下方结果区：
  - 触发结果：成功/失败消息（`Alert` 组件）
  - 快照预览：如果查询返回数据，显示快照基本信息：
    - 交易日期、快照生成时间、是否缓存命中
    - 股票总数、涨跌家数汇总
    - 可选：小型 TreeMap 预览（复用 `HeatmapTreemapChart` 组件，传入快照数据）

**交互流程**：

1. 管理员输入日期 → 点击"触发快照聚合"
2. 弹出确认对话框（`Dialog`）：`"确定要为 XXXXXXXX 触发热力图快照聚合吗？"`
3. 确认后调用 API，显示结果
4. 管理员可输入日期 → 点击"查询快照"查看历史数据
5. 查询结果展示快照元信息 + 数据预览
