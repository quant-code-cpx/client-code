## 一、模块定位与设计目标

### 1.1 为什么需要独立的策略管理模块

当前回测工作台允许用户临时配置策略参数后提交回测，但存在痛点：

| 痛点                           | 解决                                        |
| ------------------------------ | ------------------------------------------- |
| 每次回测要重新配策略参数       | 保存为策略模板，一键发起回测                |
| 策略配置无法复用               | 策略 CRUD + 克隆，反复迭代                  |
| 策略无法分享                   | `isPublic` 控制公开，其他用户可克隆公开策略 |
| 回测工作台的"草稿"只是临时存储 | 策略模板是持久化的、带版本管理的正式实体    |
| 缺乏策略全局视图               | 策略列表页按类型/标签筛选，快速找到目标策略 |

### 1.2 与回测工作台的关系

```text
策略管理模块                          回测工作台
┌──────────────┐                    ┌──────────────┐
│ 创建/编辑策略 │──── 发起回测 ────▶│ 回测参数确认  │
│ 策略列表/详情 │    (strategies/   │ 校验 → 提交   │
│ 克隆/删除     │     run)         │ 跳转详情页    │
└──────────────┘                    └──────────────┘
       │                                    │
       │                                    ▼
       │                            ┌──────────────┐
       └──── 从回测结果中 ◀─────────│ 回测详情页    │
             "保存为策略"            └──────────────┘
```

- `POST /strategies/run`：从策略管理发起回测（合并 `backtestDefaults` + 请求参数）
- `POST /backtests/runs`：从回测工作台发起回测（手动配置全部参数）
- 两者最终创建的都是 BacktestRun，区别是入口和参数来源不同

### 1.3 设计原则

1. **鼠标友好**：所有策略配置使用结构化表单（Select / Autocomplete / Slider / Switch），禁止原始 JSON textarea
2. **复用回测组件**：策略参数表单复用 `backtest-strategy-config-panel.tsx` 中的四个子面板
3. **版本可追溯**：每次 config 变更自动递增 version，UI 需显示版本号
4. **渐进增强**：先实现列表+详情+CRUD，再迭代高级功能（批量对比、公开策略市场）

## 四、后端 API 映射清单

### 4.1 Strategy 实体定义

```typescript
export type Strategy = {
  id: string; // cuid
  userId: number;
  name: string; // varchar(100), 同一用户下唯一
  description: string | null; // varchar(500)
  strategyType: string; // varchar(50), 枚举 5 种类型
  strategyConfig: Record<string, unknown>; // JSONB, 按 strategyType 对应的 JSON Schema 校验
  backtestDefaults: Record<string, unknown> | null; // JSONB, 回测默认参数
  tags: string[]; // 最多 10 个标签
  version: number; // 自增，config 变更时 +1
  isPublic: boolean;
  createdAt: string; // ISO DateTime
  updatedAt: string; // ISO DateTime
};
```

### 4.2 端点清单

| 功能        | 端点                       | 请求体                                                                                                                                                                                   | 响应体                                                                        |
| ----------- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| 创建策略    | `POST /strategies`         | `{ name: string; description?: string; strategyType: string; strategyConfig: Record<string, unknown>; backtestDefaults?: Record<string, unknown>; tags?: string[]; isPublic?: boolean }` | `Strategy`                                                                    |
| 策略列表    | `POST /strategies/list`    | `{ strategyType?: string; tags?: string[]; keyword?: string; page?: number; pageSize?: number; isPublic?: boolean }`                                                                     | `{ items: Strategy[]; total: number; page: number; pageSize: number }`        |
| 策略详情    | `POST /strategies/detail`  | `{ id: string }`                                                                                                                                                                         | `Strategy`                                                                    |
| 更新策略    | `POST /strategies/update`  | `{ id: string; name?: string; description?: string; strategyConfig?: Record<string, unknown>; backtestDefaults?: Record<string, unknown>; tags?: string[]; isPublic?: boolean }`         | `Strategy`（config 变更时 version 自动 +1）                                   |
| 删除策略    | `POST /strategies/delete`  | `{ id: string }`                                                                                                                                                                         | `{ message: string }`                                                         |
| 克隆策略    | `POST /strategies/clone`   | `{ id: string; name?: string }`                                                                                                                                                          | `Strategy`（新 id，version=1，userId 为当前用户）                             |
| 发起回测    | `POST /strategies/run`     | `{ strategyId: string; name?: string; startDate?: string; endDate?: string; initialCapital?: number; benchmarkTsCode?: string; universe?: string; rebalanceFrequency?: string; ... }`    | `{ runId: string; status: 'QUEUED'; jobId: string }`                          |
| 获取 Schema | `POST /strategies/schemas` | `{}`（空对象或无 body）                                                                                                                                                                  | `Record<string, JSONSchema>`（key 为 strategyType，value 为对应 JSON Schema） |

#### 数据加载

```typescript
useEffect(() => {
  let cancelled = false;
  const load = async () => {
    setLoading(true);
    try {
      const res = await strategyApi.listStrategies({
        strategyType: filterType || undefined,
        tags: filterTags.length > 0 ? filterTags : undefined,
        keyword: keyword || undefined,
        page,
        pageSize,
      });
      if (!cancelled) {
        setStrategies(res.items);
        setTotal(res.total);
      }
    } finally {
      if (!cancelled) setLoading(false);
    }
  };
  load();
  return () => {
    cancelled = true;
  };
}, [filterType, filterTags, keyword, page, pageSize]);
```

#### 新建策略对话框 `StrategyCreateDialog`

**交互流程**:

1. 用户点击"新建策略"
2. 弹出全屏对话框或大型 Dialog（`maxWidth="md"`）
3. 步骤式表单（非 Stepper，而是单页滚动分区）：
   - **A. 基本信息区**：名称（TextField）、描述（TextField multiline）、标签（Autocomplete freeSolo multiple，最多 10 个）、是否公开（Switch）
   - **B. 策略类型选择**：5 张卡片（类似回测工作台的模板卡片选择）
   - **C. 策略配置区**：根据选中的策略类型，动态渲染对应配置面板（复用 `BacktestStrategyConfigPanel` 或新建增强版）
   - **D. 回测默认参数区**（可选展开）：回测区间、初始资金、基准、股票池、调仓频率等
4. 底部操作：`取消` / `创建`

**表单校验**:

- 名称必填，1~100 字符
- 策略类型必选
- strategyConfig 按对应 JSON Schema 校验（前端基础校验，后端兜底）
- 标签最多 10 个

#### 克隆对话框 `StrategyCloneDialog`

**交互流程**:

1. 用户点击克隆 → 弹出小型 Dialog
2. 显示原策略名称，提供新名称输入框（默认值：`{原名称} - 副本`）
3. `取消` / `确认克隆`
4. 克隆成功后刷新列表，跳转到新策略详情页

#### 删除确认对话框 `StrategyDeleteDialog`

- 显示策略名称
- 警告文案："删除后不可恢复，关联的回测记录不会被删除"
- `取消` / `确认删除`（红色按钮）

## 九、数据加载策略

| 场景             | 策略                                                            |
| ---------------- | --------------------------------------------------------------- |
| 策略列表初始加载 | 进入页面即请求，`page=1, pageSize=12`                           |
| 筛选/搜索        | 关键词 debounce 300ms 后触发请求；类型/标签变更立即触发         |
| 分页             | 切页时请求新数据，保留筛选条件                                  |
| 策略详情         | 进入详情页即请求 `getStrategyDetail`                            |
| 策略模板 Schema  | 新建策略对话框打开时请求一次 `getStrategySchemas`，缓存到 state |
| 关联回测记录     | 详情页加载策略后，并行请求回测列表（取最近 5 条）               |
| 创建/更新/删除后 | 操作成功后刷新列表或详情                                        |
