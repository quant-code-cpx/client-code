## 一、模块定位与设计目标

### 1.1 为什么需要事件驱动研究模块

量化投资中，特定事件（业绩预告、股东增减持、限售解禁等）对股价有显著影响。投资者需要：

| 痛点                         | 解决                                             |
| ---------------------------- | ------------------------------------------------ |
| 无法量化事件对股价的影响     | 事件影响分析（AAR/CAAR 超额收益计算 + 统计检验） |
| 事件信息分散、难以系统化查阅 | 分类事件查询（8 种事件类型分页浏览）             |
| 无法基于事件自动触发交易信号 | 信号规则引擎（CRUD + 手动扫描 + 信号历史）       |
| 研究成果无法沉淀             | 信号规则持久化，历史信号可追溯                   |

### 1.2 与其他模块的关系

```text
事件驱动研究                         策略管理 / 回测
┌──────────────┐                   ┌──────────────┐
│ 事件查询/分析 │── 发现有效事件 ──▶│ 基于事件构建   │
│ 信号规则管理  │   因子/信号       │ 量化策略       │
└──────────────┘                   └──────────────┘
       │
       ▼
┌──────────────┐
│ 信号历史      │── 已触发信号 ──▶ 研究笔记 / 自选股
└──────────────┘
```

### 1.3 设计原则

1. **鼠标友好**：事件类型、参数窗口等使用 Select / Slider / DatePicker，禁止手动输入 JSON
2. **可视化优先**：AAR/CAAR 必须用折线图呈现，Top/Bottom 样本用表格展示
3. **Tab 化页面**：事件查询、事件分析、信号规则、信号历史四个 Tab 在同一页面切换
4. **渐进增强**：先实现查询+分析，再实现信号规则 CRUD

**结果渲染**（嵌套子组件）：

#### 7.3.1 `EventAnalysisSummaryCards` — 概要卡片

**Props**：`{ result: EventAnalyzeResult }`

**布局**：`Grid container spacing={3}`，4 张统计卡片，每张 `xs={6} md={3}`

| 卡片     | 值                   | 颜色规则                                             |
| -------- | -------------------- | ---------------------------------------------------- |
| 样本数量 | `result.sampleCount` | `info.main`                                          |
| CAAR     | `result.caar`        | > 0 → `success.main`，< 0 → `error.main`             |
| t 统计量 | `result.tStatistic`  | `primary.main`                                       |
| p 值     | `result.pValue`      | < 0.05 → `success.main`（显著），否则 `warning.main` |

**格式化**：

- CAAR：百分比，保留 2 位小数，带 `%` 后缀和 `+/-` 符号
- t 统计量：保留 3 位小数
- p 值：保留 4 位小数

## 九、交互细节与边界处理

### 9.1 加载与空状态

| 场景               | 处理方式                                      |
| ------------------ | --------------------------------------------- |
| 事件类型列表加载中 | Tab 标签区域显示 `Skeleton`                   |
| 查询中/分析中      | 结果区域显示 `CircularProgress` 居中          |
| 查询结果为空       | 表格区域显示空状态插图 + "暂无匹配的事件记录" |
| 分析样本太少       | 正常展示结果，`sampleCount` 卡片文字提示      |
| API 错误           | `Snackbar` 提示错误信息，结果区域保持上次数据 |

### 9.2 表单校验

| 字段     | 校验规则                          |
| -------- | --------------------------------- |
| 事件类型 | 必填，否则查询/分析按钮 disabled  |
| 日期范围 | startDate ≤ endDate（如果都填了） |
| preDays  | 0-60 整数，Slider 自动限制        |
| postDays | 1-120 整数，Slider 自动限制       |
| 规则名称 | 非空、max 128 字符                |

### 9.3 权限控制

- **手动扫描按钮**（Tab 3 顶部）：仅管理员可见，使用 `useAuthContext()` 判断 `user.role === 'SUPER_ADMIN'`
- 其他功能所有登录用户可用

### 9.4 跨模块联动（未来增强）

- 信号历史中的股票代码可跳转至 `/stock/detail?ts_code=xxx`
- 事件分析结果可一键创建研究笔记（调用 research-note API）
- Top/Bottom 样本可一键加入自选股
