## 一、设计目标

当前首页 `src/sections/overview/` 渲染的是 Minimal UI 模板自带的电商分析组件（Weekly Sales / New Users / Purchase Orders / Messages 等），全部使用 `src/_mock/` 模拟数据，与量化研究平台无关。

**目标**：将首页改造为一个量化研究快速入口仪表盘，用户登录后能在一屏内获取：

- 市场整体温度（指数、情绪、资金）
- 研究线索入口（热门板块、主力个股）
- 个人任务进展（最近回测）
- 系统状态概览（数据同步，仅管理员）

**设计原则**：

1. **仅使用现有后端 API** — 不依赖待开发接口（自选股、策略预警等留空位但不实现）
2. **最小化首次加载** — 首屏只请求核心数据，次要模块滚动/点击后按需加载
3. **复用现有组件** — 尽量复用项目内 `Chart`、`Label`、`Scrollbar` 等共享组件
4. **响应式优先** — 桌面三栏 / 平板两栏 / 手机单栏

## 三、模块详细设计

### A. 市场快照卡片（Market Index Cards）

**组件名**：`DashboardIndexCard`

| 属性       | 说明                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 指数名称   | 沪深 300 / 中证 500 / 上证指数 / 深证成指                            |
| 最新点位   | `close` 字段，保留 2 位小数                                          |
| 涨跌幅     | `pctChg` 字段，红涨绿跌，带 ▲/▼ 图标                                 |
| 涨跌额     | `change` 字段                                                        |
| 成交额     | `amount` 字段，自动换算为"亿"                                        |
| 迷你走势图 | 使用 `IndexTrend` API 获取近 5 日分时数据，渲染 ApexCharts sparkline |

**数据来源**：

- `POST /api/market/index-quote` → `IndexQuoteItem[]`
- `POST /api/market/index-trend` → 补充 sparkline 数据

**交互**：

- 点击卡片跳转 `/market/overview`
- 鼠标悬停显示"查看详情"提示

**布局**：`Grid size={{ xs: 12, sm: 6, md: 3 }}`（一行 4 卡片，平板 2×2，手机纵向排列）

---

### B. 市场情绪区（Market Sentiment）

#### B1. 情绪概览卡（Sentiment Summary Card）

**组件名**：`DashboardSentimentCard`

| 展示项     | 数据字段                | 说明                             |
| ---------- | ----------------------- | -------------------------------- |
| 上涨家数   | `riseCount`             | 红色数字                         |
| 下跌家数   | `fallCount`             | 绿色数字                         |
| 平盘家数   | `flatCount`             | 灰色数字                         |
| 涨停家数   | `limitUpCount`          | 红色 Label 标签                  |
| 跌停家数   | `limitDownCount`        | 绿色 Label 标签                  |
| 红绿比     | `riseCount / fallCount` | 进度条可视化                     |
| 涨跌比例条 | 横向堆叠条              | 红/灰/绿三段分别代表涨/平/跌占比 |

**数据来源**：`POST /api/market/sentiment` → `SentimentResult`

**布局**：`Grid size={{ xs: 12, md: 5 }}`

#### B2. 涨跌幅分布直方图（Change Distribution Chart）

**组件名**：`DashboardChangeDistribution`

- X 轴：涨跌幅区间（如 -10%以下, -7%~-5%, …, +10%以上）
- Y 轴：股票数量
- 颜色：负区间绿色填充，正区间红色填充
- 图表类型：ApexCharts `bar`

**数据来源**：`POST /api/market/change-distribution` → `ChangeDistributionResult`

**布局**：`Grid size={{ xs: 12, md: 7 }}`

---

### C. 资金与板块区（Capital Flow & Sectors）

#### C1. 资金流向概览（Money Flow Overview）

**组件名**：`DashboardMoneyFlow`

- 展示当日主力（超大单+大单）/ 中单 / 散户（小单）净流入金额
- 用横向堆叠条或双色柱状图展示流入/流出对比
- 辅以"总成交额"和"主力净占比"指标
- 金额显示单位自动切换：万元 / 亿元

**数据来源**：`POST /api/market/money-flow` → `MarketMoneyFlowDetail`

**交互**：点击"查看详情"跳转 `/market/money-flow`

**布局**：`Grid size={{ xs: 12, md: 4 }}`

#### C2. 北向资金走势（HSGT Flow Trend）

**组件名**：`DashboardHsgtFlow`

- 折线图展示近 30 日北向资金净买入
- 双轴：左轴净买入金额（柱+线）、右轴累计净买入
- 正值红色、负值绿色

**数据来源**：`POST /api/market/hsgt-flow` → `HsgtFlowHistory`（含 `days: 30`）

**布局**：`Grid size={{ xs: 12, md: 4 }}`

#### C3. 热门板块 Top 10（Hot Sectors）

**组件名**：`DashboardHotSectors`

- 表格展示今日涨幅前 10 板块
- 列：板块名、涨跌幅（%）、主力净流入（亿）、领涨股
- 涨为红色，跌为绿色
- 切换 Tab：按涨幅排序 / 按资金净流入排序

**数据来源**：

- `POST /api/market/sector-ranking` → `{ sectors: SectorRankingItem[] }`（`sort_by: 'change'`，`limit: 10`）
- `POST /api/market/sector-flow-ranking` → `SectorFlowRankingItem[]`（`sort_by: 'net_amount'`，`limit: 10`）

**交互**：点击板块名可跳转 `/market/overview`（带板块筛选参数，如有）

**布局**：`Grid size={{ xs: 12, md: 4 }}`

---

### D. 主力动向 & 最近回测

#### D1. 主力净流入 Top 10（Main Flow Ranking）

**组件名**：`DashboardMainFlowRanking`

- 表格展示今日主力净流入前 10 个股
- 列：股票代码、名称、最新价、涨跌幅（%）、主力净流入（亿）
- 支持切换"净流入"/"净流出"视角
- 涨跌幅和净流入带颜色标识

**数据来源**：`POST /api/market/main-flow-ranking` → `{ data: MainFlowRankingItem[] }`（`limit: 10`，`order: 'desc'`）

**交互**：点击股票代码跳转 `/stock/detail?code=xxx`

**布局**：`Grid size={{ xs: 12, md: 7 }}`

#### D2. 最近回测任务（Recent Backtests）

**组件名**：`DashboardRecentBacktests`

- 卡片列表展示最近 5 条回测任务
- 每张卡片包含：
  - 策略名称
  - 状态 `Label`（COMPLETED → success / RUNNING → info / FAILED → error / QUEUED → warning）
  - 核心指标：年化收益率、最大回撤、夏普比率
  - 创建时间（相对时间，如"3 小时前"）
- 空状态：显示"暂无回测任务"提示和"前往创建"按钮

**数据来源**：`GET /api/backtests/runs` → `BacktestRunListResponse`（`page: 1`，`pageSize: 5`）

**交互**：

- 点击卡片跳转 `/backtest/runs/:runId`
- "查看全部"按钮跳转 `/backtest/runs`
- "新建回测"按钮跳转 `/backtest`

**布局**：`Grid size={{ xs: 12, md: 5 }}`

---

### E. 系统状态（System Status） — 仅管理员

**组件名**：`DashboardSystemStatus`

- 仅 `SUPER_ADMIN` / `ADMIN` 角色可见（使用 `usePermission` 判断）
- 表格展示数据同步计划列表
- 列：任务名称、是否启用、Cron 表达式、最后成功时间、下次执行时间
- 用绿色/灰色圆点表示启用/禁用状态
- 可折叠（默认折叠，标题显示"数据同步状态"）

**数据来源**：`GET /api/tushare/admin/plans` → `TushareSyncPlan[]`

**布局**：`Grid size={{ xs: 12 }}`（全宽，位于页面底部）

## 五、数据加载策略

### 5.1 首屏关键数据（并行加载）

首页挂载时，使用 `Promise.all` 并行拉取以下 API（或封装为自定义 Hook）：

```
fetchIndexQuote()          → 指数卡片
fetchSentiment()           → 情绪概览
fetchChangeDistribution()  → 涨跌分布
fetchMoneyFlow()           → 资金概览
```

这 4 个接口的数据量小（< 5KB each），延迟低，适合首屏加载。

### 5.2 次屏延迟加载

以下数据在用户滚动到对应区域或页面 idle 后加载：

```
fetchIndexTrend()              → 指数 sparkline（可 idle 时预取）
fetchHsgtFlow({ days: 30 })    → 北向资金走势
fetchSectorRanking({ limit: 10 })  → 板块排名
fetchMainFlowRanking({ limit: 10 })→ 主力排名
backtestApi.listRuns({ pageSize: 5 }) → 最近回测
```

### 5.3 条件加载

```
tushareSyncApi.getPlans()  → 仅管理员角色时加载
```

### 5.4 自动刷新

- 盘中时段（9:15 ~ 15:15）每 60 秒自动刷新市场快照和情绪数据
- 使用 `setInterval` + `AbortController` 管理生命周期
- 非盘中时段不自动刷新，减少无意义请求

### 5.5 错误处理

- 每个模块独立加载，单个 API 失败不影响其他模块渲染
- 失败模块显示 Skeleton 占位 + "加载失败，点击重试" 提示
- 不使用全局 loading 遮罩

## 附录：颜色约定

| 语义      | 色值来源                 | 用途                   |
| --------- | ------------------------ | ---------------------- |
| 上涨/正值 | `error.main`（红色）     | 指数涨、净流入正、涨停 |
| 下跌/负值 | `success.main`（绿色）   | 指数跌、净流出、跌停   |
| 平盘/中性 | `text.secondary`（灰色） | 平盘家数、无变化       |
| 主力      | `primary.main`           | 超大单/大单标识        |
| 散户      | `warning.main`           | 小单/中单标识          |

> **注意**：中国 A 股市场采用"红涨绿跌"色彩惯例，与欧美相反。本项目使用 MUI 的 `error`（红）代表涨、`success`（绿）代表跌。
