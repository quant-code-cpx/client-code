```typescript
async function handleSubmit() {
  setLoading(true);
  try {
    let portfolioId = selectedPortfolioId;
    // 如果选择新建组合，先创建
    if (createNew) {
      const created = await createPortfolio({
        name: newName,
        initialCash: newInitialCash,
      });
      portfolioId = created.id;
    }
    // 调用导入接口
    await applyBacktest({ portfolioId, runId, mode });
    onSuccess();
    onClose();
  } finally {
    setLoading(false);
  }
}
```

**导入模式说明**：

- `REPLACE`：替换模式，会清空原组合持仓后全量导入回测末日持仓
- `MERGE`：合并模式，保留原持仓不变，对同一股票代码累加仓位

### 3.1 API 补充：Monte Carlo 蒙特卡洛模拟

**缺失端点**：`POST /backtests/runs/monte-carlo`

在 `src/api/backtest.ts` 末尾追加：

```typescript
// ─── 蒙特卡洛模拟类型 ──────────────────────────────

export type MonteCarloRequest = {
  runId: string;
  simulations?: number;       // 模拟次数，默认 1000
  confidenceLevels?: number[]; // 置信区间，默认 [0.05, 0.25, 0.75, 0.95]
};

export type MonteCarloPathPoint = {
  day: number;                 // 第 N 个交易日
  median: number;              // 中位数路径净值
  mean: number;                // 均值路径净值
  p5: number;                  // 5% 分位
  p25: number;                 // 25% 分位
  p75: number;                 // 75% 分位
  p95: number;                 // 95% 分位
};

export type MonteCarloResponse = {
  runId: string;
  simulations: number;
  stats: {
    expectedReturn: number;    // 期望年化收益
    expectedVolatility: number; // 期望年化波动率
    var95: number;             // 95% VaR（最大损失）
    cvar95: number;            // 95% CVaR（尾部平均损失）
    worstDrawdown: number;     // 最差路径最大回撤
    bestReturn: number;        // 最优路径年化收益
    worstReturn: number;       // 最差路径年化收益
    probPositive: number;      // 正收益概率
  };
  paths: MonteCarloPathPoint[]; // 分位数包络线数据
};

// ─── 蒙特卡洛模拟 API ─────────────────────────────

export function runMonteCarlo(query: MonteCarloRequest) {
  return apiClient.post<MonteCarloResponse>('/api/backtests/runs/monte-carlo', query);
}
```

**两步式交互流程**：

1. **提交扫描**：用户配置参数搜索空间 → 调用 `createParamSensitivity()` → 获得 `{ jobId, status }`
2. **轮询/等待结果**：通过 WebSocket 监听 `jobId` 完成事件 → 调用 `getParamSensitivityResult()` → 获取热力图数据

```typescript
// 步骤 1
const createRes = await createParamSensitivity({
  runId,
  paramSpace: paramSearchSpace,   // Record<string, ParamSearchSpaceItem>
  metric: 'sharpeRatio',
});

// 步骤 2（jobId 完成后）
const result = await getParamSensitivityResult({ jobId: createRes.jobId });
// result.cells: ParamSensitivityCell[]
// 每个 cell: { params: Record<string, unknown>; metrics: { annualizedReturn, sharpeRatio, maxDrawdown } }
```

**热力图实现**：

- 仅当恰好 2 个参数时渲染热力图

- X 轴 = 第一个参数值序列，Y 轴 = 第二个参数值序列
- 单元格颜色根据选择的优化指标（夏普/年化/回撤）着色

- 点击单元格可高亮对应表格行

## 附录：UI 规范速查

| 元素           | 实现方式                                                          |
| -------------- | ----------------------------------------------------------------- |
| 正值/负值颜色  | 正 → `success.main`，负 → `error.main`                           |
| 摘要卡片       | `Grid` xs=6 sm=3，Card + Typography variant="h4" / "caption"     |
| 状态标签       | `Label` 组件 + color mapping                                     |
| 数值格式化     | 收益率 → `(x * 100).toFixed(2) + '%'`；比率 → `x.toFixed(2)`    |
| 图表           | ApexCharts + `useChart` hook                                      |
| 日期显示       | 统一 `YYYY-MM-DD` 格式                                           |
| 半透明填充色   | `varAlpha(theme.vars.palette.xxx.mainChannel, opacity)`           |
| 表格排序       | MUI Table + 手动排序 state                                       |
| 对话框         | MUI Dialog，宽度 `maxWidth="sm"` 或 `"md"`                       |
| 表单           | 结构化控件（Select/TextField/Switch），禁止原始 JSON 输入         |
