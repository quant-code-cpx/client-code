## 一、模块定位与设计目标

### 1.1 为什么需要版本管理

当前策略详情页展示 `version` 数字但无法回溯历史配置。每次修改 `strategyConfig` 后端自动递增版本号，但用户无从得知：

| 痛点                     | 解决                                                   |
| ------------------------ | ------------------------------------------------------ |
| 不知道改了什么           | 版本列表展示每次变更的 changelog 和时间                |
| 无法对比新旧配置         | 两版本 Diff 对比，逐字段高亮 ADDED / REMOVED / CHANGED |
| 无法追溯历史配置         | 版本列表展示每个版本的完整 `strategyConfig`            |
| 回测结果与哪个版本对应？ | 版本号 + 时间戳帮助用户关联回测历史                    |

### 1.2 与现有策略详情页的关系

```text
现有策略详情页                          增强后策略详情页
┌──────────────────────────┐          ┌──────────────────────────────────┐
│ StrategyDetailHeader     │          │ StrategyDetailHeader             │
│ ┌─ Left ──┐ ┌─ Right ──┐│          │ ┌─ Left ──┐ ┌─ Right ──────────┐│
│ │InfoCard  │ │SignalCard││          │ │InfoCard  │ │SignalCard        ││
│ │ConfigCard│ │QuickRun  ││          │ │ConfigCard│ │QuickRun          ││
│ │Backtest  │ │RunHistory││          │ │Backtest  │ │RunHistory        ││
│ │Defaults  │ │          ││  ──▶    │ │Defaults  │ │VersionHistory   ││
│ └──────────┘ └──────────┘│          │ └──────────┘ └─────────────────┘│
└──────────────────────────┘          └──────────────────────────────────┘
```

在右侧列底部新增 **StrategyVersionCard**，不影响现有布局。

### 1.3 设计原则

1. **不新增路由**：版本管理嵌入策略详情页右侧列，不增加独立页面
2. **只读浏览**：版本历史为只读，不支持"回滚到历史版本"（后端未提供回滚接口）
3. **Diff 可视化**：版本对比使用表格式 Diff，逐行显示 path / oldValue / newValue / changeType
4. **按需加载**：版本列表和 Diff 数据均在用户交互时按需请求

## 四、后端 API 映射清单

### 4.1 已有类型定义（`src/api/strategy.ts`）

```typescript
export type StrategyVersionItem = {
  version: number;
  strategyConfig: Record<string, unknown>;
  backtestDefaults: Record<string, unknown> | null;
  changelog: string | null;
  createdAt: string;
  isCurrent: boolean;
};

export type ConfigDiffItem = {
  path: string; // JSON path，如 "shortPeriod"、"tsCode"
  oldValue: unknown;
  newValue: unknown;
  changeType: 'ADDED' | 'REMOVED' | 'CHANGED';
};

export type CompareVersionsRequest = {
  strategyId: string;
  versionA: number;
  versionB: number;
};

export type CompareVersionsResponse = {
  strategyId: string;
  versionA: StrategyVersionItem;
  versionB: StrategyVersionItem;
  configDiff: ConfigDiffItem[];
};
```

### 4.2 已有 API 方法

| 功能     | 端点                                | 方法                           |
| -------- | ----------------------------------- | ------------------------------ |
| 版本列表 | `POST /strategies/versions`         | `listStrategyVersions(id)`     |
| 版本对比 | `POST /strategies/compare-versions` | `compareStrategyVersions(dto)` |

### 4.3 业务约束

| 约束         | 规则                                             |
| ------------ | ------------------------------------------------ |
| 版本自增规则 | 仅当 `strategyConfig` 变更时 version + 1         |
| 版本不可删除 | 后端不提供删除历史版本接口                       |
| 不可回滚     | 后端不提供"恢复到指定版本"接口                   |
| 对比范围     | 只能对比同一策略的两个版本                       |
| Diff 内容    | 仅对比 `strategyConfig`，不含 `backtestDefaults` |

### 7.2 StrategyVersionDiffDialog

**Props**

```typescript
type StrategyVersionDiffDialogProps = {
  open: boolean;
  onClose: () => void;
  strategyId: string;
  versionA: number;
  versionB: number;
};
```

**内部状态**

```typescript
const [diffData, setDiffData] = useState<CompareVersionsResponse | null>(null);
const [loading, setLoading] = useState(false);
const [error, setError] = useState('');
```

**数据加载**

```typescript
useEffect(() => {
  if (!open) return;
  setLoading(true);
  setError('');
  compareStrategyVersions({ strategyId, versionA, versionB })
    .then(setDiffData)
    .catch((err) => setError(err instanceof Error ? err.message : '获取对比数据失败'))
    .finally(() => setLoading(false));
}, [open, strategyId, versionA, versionB]);
```

**Diff 表格渲染**

- changeType 映射：
  - `'ADDED'` → `Label color="success"` 文本"新增"
  - `'REMOVED'` → `Label color="error"` 文本"删除"
  - `'CHANGED'` → `Label color="warning"` 文本"修改"
- `oldValue` / `newValue` 使用 `JSON.stringify(value, null, 2)` 显示，值为 `undefined` 时显示 `—`
- 空 Diff（`configDiff.length === 0`）时显示"两个版本配置完全一致"

**UI 框架**

```tsx
<Dialog open={open} onClose={onClose} maxWidth="md" fullWidth>
  <DialogTitle>
    版本对比 v{versionA} → v{versionB}
  </DialogTitle>
  <DialogContent>
    {loading && <Skeleton variant="rectangular" height={200} />}

    {error && <Alert severity="error">{error}</Alert>}

    {diffData && (
      <>
        {/* 版本信息摘要 */}
        <Box sx={{ display: 'flex', gap: 2, mb: 3 }}>
          <Box sx={{ flex: 1, p: 1.5, borderRadius: 1, bgcolor: 'background.neutral' }}>
            <Typography variant="subtitle2">版本 {diffData.versionA.version}</Typography>
            <Typography variant="caption" color="text.secondary">
              {fDateTime(diffData.versionA.createdAt)}
            </Typography>
          </Box>
          <Box sx={{ flex: 1, p: 1.5, borderRadius: 1, bgcolor: 'background.neutral' }}>
            <Typography variant="subtitle2">版本 {diffData.versionB.version}</Typography>
            <Typography variant="caption" color="text.secondary">
              {fDateTime(diffData.versionB.createdAt)}
            </Typography>
          </Box>
        </Box>

        {/* Diff 表格 */}
        {diffData.configDiff.length === 0 ? (
          <Box sx={{ py: 4, textAlign: 'center' }}>
            <Typography color="text.secondary">两个版本配置完全一致</Typography>
          </Box>
        ) : (
          <Table size="small">
            <TableHead>
              <TableRow>
                <TableCell>配置路径</TableCell>
                <TableCell>旧值 (v{versionA})</TableCell>
                <TableCell>新值 (v{versionB})</TableCell>
                <TableCell align="center">变更类型</TableCell>
              </TableRow>
            </TableHead>
            <TableBody>
              {diffData.configDiff.map((item, index) => (
                <TableRow key={index}>
                  <TableCell>
                    <Typography variant="body2" sx={{ fontFamily: 'monospace' }}>
                      {item.path}
                    </Typography>
                  </TableCell>
                  <TableCell>
                    <Typography variant="body2" sx={{ fontFamily: 'monospace' }}>
                      {item.oldValue !== undefined ? JSON.stringify(item.oldValue) : '—'}
                    </Typography>
                  </TableCell>
                  <TableCell>
                    <Typography variant="body2" sx={{ fontFamily: 'monospace' }}>
                      {item.newValue !== undefined ? JSON.stringify(item.newValue) : '—'}
                    </Typography>
                  </TableCell>
                  <TableCell align="center">
                    <Label color={DIFF_LABEL_MAP[item.changeType].color}>
                      {DIFF_LABEL_MAP[item.changeType].text}
                    </Label>
                  </TableCell>
                </TableRow>
              ))}
            </TableBody>
          </Table>
        )}
      </>
    )}
  </DialogContent>
  <DialogActions>
    <Button onClick={onClose}>关闭</Button>
  </DialogActions>
</Dialog>
```

**常量**

  REMOVED: { color: 'error', text: '删除' },

## 十、错误处理与边界情况

| 场景              | 处理方式                                        |
| ----------------- | ----------------------------------------------- |
| 只有 v1（无历史） | 版本列表正常显示 v1，隐藏"对比"按钮和底部选择器 |
| 网络错误          | 版本列表区域显示 Alert error                    |
| 对比加载失败      | Dialog 内显示 Alert error                       |
| Diff 结果为空     | 显示"两个版本配置完全一致"空状态                |
| 选择相同版本对比  | "对比"按钮 disabled                             |
