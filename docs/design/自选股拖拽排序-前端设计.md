## 一、模块定位与设计目标

### 1.1 为什么需要拖拽排序

当前自选组内股票按 `sortOrder` 字段排序，但用户无法在前端调整顺序。

| 痛点                         | 解决                           |
| ---------------------------- | ------------------------------ |
| 股票列表顺序固定，无法自定义 | 拖拽行改变顺序，松手后自动保存 |
| 重要关注股无法置顶           | 拖拽到最前即可置顶             |
| 批量添加后顺序不理想         | 随时拖拽微调                   |

### 1.2 与现有页面的关系

```text
现有 WatchlistStockTable                增强后 WatchlistStockTable
┌───────────────────────────────┐      ┌────────────────────────────────────┐
│ ☐ │ 名称/代码 │ 现价 │ ...   │      │ ⠿ │ ☐ │ 名称/代码 │ 现价 │ ...   │
│ ☐ │ 平安银行  │ 12.5 │ ...   │      │ ⠿ │ ☐ │ 平安银行  │ 12.5 │ ...   │
│ ☐ │ 招商银行  │ 38.2 │ ...   │ ──▶ │ ⠿ │ ☐ │ 招商银行  │ 38.2 │ ...   │
│ ☐ │ 贵州茅台  │ 1680 │ ...   │      │ ⠿ │ ☐ │ 贵州茅台  │ 1680 │ ...   │
└───────────────────────────────┘      └────────────────────────────────────┘
                                        ↑ 新增拖拽手柄列
```

在表格最左侧（Checkbox 之前）新增拖拽手柄列，支持行级拖拽排序。

### 1.3 设计原则

1. **最小改动**：仅在现有表格中新增拖拽能力，不改变表格结构
2. **手柄触发**：通过拖拽手柄（⠿ 图标）触发，避免干扰其他点击交互
3. **乐观更新**：拖拽松手后立即更新 UI 顺序，后台异步保存
4. **失败回滚**：若保存失败，回滚到拖拽前的顺序并 toast 提示

| ------------ | -------------------------------- | -------------------------------------------------------------------------- | --------------------- |
| 批量更新排序 | `POST /watchlist/stocks/reorder` | `{ watchlistId: number; items: Array<{ id: number; sortOrder: number }> }` | `{ message: string }` |

### 3.2 API 方法

在 `src/api/watchlist.ts` 中新增：

```typescript
export function reorderStocks(
  watchlistId: number,
  items: Array<{ id: number; sortOrder: number }>
) {
  return apiClient.post<{ message: string }>('/api/watchlist/stocks/reorder', {
    watchlistId,
    items,
  });
}
```

### 3.3 业务约束

| 约束           | 规则                                   |
| -------------- | -------------------------------------- |
| 权限           | 只能对自己的自选组排序                 |
| sortOrder 范围 | 从 0 开始的连续整数                    |
| 幂等性         | 多次提交相同排序结果不报错             |
| 部分排序       | 可只提交变动项，但建议全量提交避免冲突 |

## 六、组件详细设计

### 6.1 WatchlistStockTable — 拖拽容器增强

**新增 Props**

```typescript
type WatchlistStockTableProps = {
  // ... 现有 props 不变
  stocks: WatchlistStock[];
  loading: boolean;
  selectedIds: number[];
  onSelectAll: (checked: boolean) => void;
  onSelect: (id: number) => void;
  onEdit: (row: WatchlistStock) => void;
  onRemove: (id: number) => void;
  // 新增
  onReorder: (reorderedStocks: WatchlistStock[]) => void;
};
```

**拖拽容器实现**

```tsx
import { DndContext, closestCenter, KeyboardSensor, PointerSensor, useSensor, useSensors } from '@dnd-kit/core';
import type { DragEndEvent } from '@dnd-kit/core';
import { SortableContext, verticalListSortingStrategy, arrayMove } from '@dnd-kit/sortable';
import { restrictToVerticalAxis } from '@dnd-kit/modifiers';

export function WatchlistStockTable({
  stocks,
  loading,
  selectedIds,
  onSelectAll,
  onSelect,
  onEdit,
  onRemove,
  onReorder,
}: WatchlistStockTableProps) {
  const sensors = useSensors(
    useSensor(PointerSensor, { activationConstraint: { distance: 5 } }),
    useSensor(KeyboardSensor)
  );

  const handleDragEnd = useCallback((event: DragEndEvent) => {
    const { active, over } = event;
    if (!over || active.id === over.id) return;

    const oldIndex = stocks.findIndex((s) => s.id === active.id);
    const newIndex = stocks.findIndex((s) => s.id === over.id);
    if (oldIndex === -1 || newIndex === -1) return;

    const reordered = arrayMove(stocks, oldIndex, newIndex);
    onReorder(reordered);
  }, [stocks, onReorder]);

  const stockIds = useMemo(() => stocks.map((s) => s.id), [stocks]);

  return (
    <DndContext
      sensors={sensors}
      collisionDetection={closestCenter}
      modifiers={[restrictToVerticalAxis]}
      onDragEnd={handleDragEnd}
    >
      <Scrollbar>
        <TableContainer sx={{ overflow: 'unset' }}>
          <Table sx={{ minWidth: 840 }}>
            <TableHead>
              <TableRow>
                <TableCell sx={{ width: 40, p: 0 }} /> {/* 拖拽手柄列 */}
                <TableCell padding="checkbox" sx={{ width: 80 }}>
                  <Checkbox
                    checked={allSelected}
                    indeterminate={indeterminate}
                    onChange={(e) => onSelectAll(e.target.checked)}
                    size="small"
                  />
                </TableCell>
                <TableCell>名称/代码</TableCell>
                <TableCell align="right">现价</TableCell>
                <TableCell align="right">涨跌幅</TableCell>
                <TableCell align="right">成交量</TableCell>
                <TableCell align="right">PE</TableCell>
                <TableCell align="right">PB</TableCell>
                <TableCell align="right">目标价</TableCell>
                <TableCell align="right">操作</TableCell>
              </TableRow>
            </TableHead>

            <TableBody>
              <SortableContext items={stockIds} strategy={verticalListSortingStrategy}>
                {loading
                  ? /* skeleton rows */
                  : stocks.map((stock) => (
                      <SortableStockRow
                        key={stock.id}
                        stock={stock}
                        selected={selectedIds.includes(stock.id)}
                        onSelect={() => onSelect(stock.id)}
                        onEdit={() => onEdit(stock)}
                        onRemove={() => onRemove(stock.id)}
                      />
                    ))
                }
              </SortableContext>
            </TableBody>
          </Table>
        </TableContainer>
      </Scrollbar>
    </DndContext>
  );
}
```

### 6.2 SortableStockRow — 可拖拽行

在 `watchlist-stock-table-row.tsx` 中增强：

```tsx
import { useSortable } from '@dnd-kit/sortable';
import { CSS } from '@dnd-kit/utilities';

export function SortableStockRow({ stock, selected, onSelect, onEdit, onRemove }: Props) {
  const { attributes, listeners, setNodeRef, transform, transition, isDragging } = useSortable({
    id: stock.id,
  });

  const style = {
    transform: CSS.Transform.toString(transform),
    transition,
    opacity: isDragging ? 0.5 : 1,
    background: isDragging ? 'rgba(0,0,0,0.04)' : undefined,
  };

  return (
    <TableRow ref={setNodeRef} style={style} hover>
      {/* 拖拽手柄 */}
      <TableCell sx={{ width: 40, p: 0, cursor: 'grab' }} {...attributes} {...listeners}>
        <Box sx={{ display: 'flex', justifyContent: 'center' }}>
          <Iconify icon="solar:hamburger-menu-bold" sx={{ color: 'text.disabled' }} />
        </Box>
      </TableCell>
      {/* Checkbox */}
      <TableCell padding="checkbox">
        <Checkbox checked={selected} onChange={onSelect} size="small" />
      </TableCell>
      {/* 其余列保持不变，复用现有 WatchlistStockTableRow 的渲染逻辑 */}
      ...
    </TableRow>
  );
}
```

> **实现方式选择**：可以将 `SortableStockRow` 作为 `WatchlistStockTableRow` 的包装组件，也可以直接在 `WatchlistStockTableRow` 中集成 `useSortable`。推荐直接集成以减少组件层级。

### 6.3 WatchlistDetailPanel — 排序回调处理

在 `watchlist-detail-panel.tsx` 中新增排序处理逻辑：

```typescript
import { reorderStocks } from 'src/api/watchlist';

// 乐观更新 + 失败回滚
const handleReorder = useCallback(
  async (reorderedStocks: WatchlistStock[]) => {
    const previousStocks = [...stocks]; // 保存旧顺序用于回滚
    setStocks(reorderedStocks); // 乐观更新 UI

    const items = reorderedStocks.map((s, index) => ({
      id: s.id,
      sortOrder: index,
    }));

    try {
      await reorderStocks(watchlistId, items);
    } catch (err) {
      setStocks(previousStocks); // 回滚
      // toast 提示保存失败
      console.error('排序保存失败', err);
    }
  },
  [stocks, watchlistId]
);
```

传入 `WatchlistStockTable`：

```tsx
<WatchlistStockTable
  stocks={stocks}
  loading={loading}
  selectedIds={selectedIds}
  onSelectAll={handleSelectAll}
  onSelect={handleSelect}
  onEdit={handleEdit}
  onRemove={handleRemove}
  onReorder={handleReorder} // 新增
/>
```

## 七、交互流程

### 7.1 拖拽排序

```text
用户 → 按住某行的拖拽手柄 ⠿
  → 开始拖拽（PointerSensor, distance > 5px 触发）
  → 行跟随鼠标移动，半透明状态
  → 拖到新位置松手
  → DndContext.onDragEnd 触发
  → arrayMove 计算新顺序
  → onReorder(reorderedStocks) 回调
  → UI 立即更新（乐观更新）
  → 后台调用 reorderStocks API
  → 成功：静默完成
  → 失败：回滚到原顺序 + toast 提示
```

### 7.2 键盘操作

```text
用户 → 聚焦到拖拽手柄
  → 按 Space 开始拖拽
  → 上下方向键移动位置
  → Enter 确认 / Esc 取消
```

## 八、错误处理与边界情况

| 场景                 | 处理方式                       |
| -------------------- | ------------------------------ |
| 空列表               | 无拖拽手柄，不渲染 DndContext  |
| 单条记录             | 显示手柄但无实际拖拽效果       |
| API 保存失败         | 回滚 UI 顺序 + console.error   |
| 拖拽中选中状态保持   | 拖拽不影响 Checkbox 选中状态   |
| 搜索/过滤后的列表    | 拖拽排序仅对当前可见列表生效   |
| 并发编辑（多标签页） | 以最后保存为准，无实时冲突处理 |
