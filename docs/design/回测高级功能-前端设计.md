### 5.2 页面二：Walk-Forward / 滚动窗口创建 `/backtest/walk-forward/create`

#### 模式区分

通过 URL 参数 `?mode=walk-forward | rolling` 控制表单差异：

| 字段       | Walk-Forward 模式      | 滚动窗口模式                         |
| ---------- | ---------------------- | ------------------------------------ |
| 样本内天数 | `inSampleDays` 显示    | 替换为 `lookbackDays`                |
| 样本外天数 | `outOfSampleDays` 显示 | 替换为 `holdingPeriodDays`           |
| 步进天数   | `stepDays` 显示        | 隐藏（后端自动 = holdingPeriodDays） |

#### 参数搜索空间编辑器（核心交互组件）

这是 Walk-Forward 区别于普通回测的核心：用户需要定义哪些策略参数需要"网格搜索"。

**组件名**：`WalkForwardParamSpaceEditor`

**交互**：

- 左侧下拉选择参数名（从策略模板的 `parameterSchema` 中获取可选字段）
- 右侧根据搜索类型（range / enum）动态切换输入：
  - **range**：显示 `min`、`max`、`step` 三个数字输入
  - **enum**：显示 Chip 输入，可添加多个枚举值
- 有 **删除** 按钮移除某一行
- 有 **添加** 按钮增加新搜索项
- 右上角显示预估组合数 = 各参数候选值数量的乘积（后端限制最大 1000）

#### 提交流程

1. 用户填写表单 → 点击"提交验证任务"
2. 前端调用 `createWalkForwardRun()` 或 `createRollingBacktest()`（根据 mode）
3. 拿到 `{ wfRunId, jobId }` 后跳转 `/backtest/walk-forward/:wfRunId`

#### OOS 净值曲线渲染

- 使用 ApexCharts **area** 图
- X 轴为 `tradeDate`
- Y 轴为 `nav`
- 按 `windowIndex` 给不同区段使用不同背景色 annotation，让用户直观看到每个 OOS 窗口的边界
- 可参考现有 `BacktestEquityChart` 组件，衍生为 `WalkForwardEquityChart`

#### 窗口明细表格

- 使用 MUI Table
- 列：窗口序号 / IS区间 / OOS区间 / 最优参数（JSON 折叠展示）/ IS收益 / IS夏普 / OOS收益 / OOS夏普 / OOS最大回撤
- 可排序列：OOS收益、OOS夏普
- 最优参数用 `Chip` 或 `Tooltip` 展示，避免表格过宽

```text
│ │                                              [删除]    │       │
│ ├───────────────────────────────────────────────────────┤       │
│ │                     [+ 添加策略]                        │       │
│ └───────────────────────────────────────────────────────┘       │
├─────────────────────────────────────────────────────────────────┤
│                                              [提交对比回测]      │
└─────────────────────────────────────────────────────────────────┘
```

#### 交互说明

- 初始默认 2 个策略卡片
- 点击 **添加策略** 增加一行，最多 10 个，满 10 个后禁用
- 每个策略卡片有 **删除** 按钮，最少保留 2 个
- 策略类型切换时，策略参数区域动态展示（复用回测工作台的 `BacktestStrategyConfigPanel`）
- 标签输入为可选文本，默认为 `策略#1`、`策略#2`...
- **调仓频率** 可按策略独立设置

#### 提交流程

1. 组装 `CreateComparisonQuery`，`strategies` 数组包含每个卡片的配置
2. 调用 `createComparison()`
3. 拿到 `{ groupId, jobId }` 后跳转 `/backtest/comparison/:groupId`
