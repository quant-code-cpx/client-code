## 一、改动范围总览

`同步管理-前端设计.md` 的基础版 DataQualityTab 只覆盖了最基础的质量检查报告、数据缺口查询和校验异常日志。随着后端 Phase 1-4 的落地，DataQualityTab 需要补充以下内容：

| 改动项                                                     | 来源 Phase | 性质 |

| 跨表对账面板                                               | Phase 3    | 新增 |

## 三、WebSocket 事件新增

在 `src/contexts/SyncNotificationContext.tsx`（或等效的 WebSocket 上下文）中补充两个事件监听：

```typescript
// 新增事件类型
type SyncSocketEvents =
  | 'sync_started'
  | 'sync_completed'
  | 'sync_error'
  | 'data_quality_completed' // ★ Phase 4 新增
  | 'auto_repair_queued'; // ★ Phase 4 新增

// 在 useEffect 中注册监听
socket.on('data_quality_completed', (summary: QualityCheckSummary) => {
  setLastQualitySummary(summary);
  // 若有 fail 项，弹出 warning 通知
  if (summary.counts.fail > 0) {
    enqueueSnackbar(`数据质量检查完成：${summary.counts.fail} 个数据集存在问题`, {
      variant: 'warning',
    });
  } else {
    enqueueSnackbar('数据质量检查完成，全部通过', { variant: 'success' });
  }
});

socket.on('auto_repair_queued', (repair: RepairSummary) => {
  if (repair.executed > 0) {
    enqueueSnackbar(`已自动入队 ${repair.executed} 个补数任务`, { variant: 'info' });
  }
});
```

const handleTriggerRepair = async () => {
  // 二次确认，防误触
  if (
    !confirm('将根据最近一轮质量检查结果，对 fail 状态的数据集自动生成补数任务并入队，是否继续？')

};

const fetchRepairStatus = async () => {
  setRepairStatusLoading(true);
