## 一、模块定位与设计目标

### 1.1 为什么需要同步状态总览

当前同步管理页有 5 个 Tab（同步计划 / 同步日志 / 数据质量 / 缓存统计 / 重试队列），但缺少一个**全局状态概览面板**让运维人员一眼掌握整体同步健康度。

| 痛点                            | 解决                                             |
| ------------------------------- | ------------------------------------------------ |
| 需要逐 Tab 查看才能了解整体状态 | 页面顶部增加状态总览卡片组，一眼看到全局健康状态 |
| 不知道哪些类别连续失败          | 按类别聚合，展示失败/成功/跳过数量               |
| 不知道上次全量同步时间          | 展示最近一次同步的时间和完成状态                 |

### 1.2 与现有页面的关系

```text
现有 TushareSyncView                    增强后 TushareSyncView
┌───────────────────────────────┐      ┌────────────────────────────────────┐
│ 标题：数据同步                 │      │ 标题：数据同步                      │
│ [计划] [日志] [质量] [缓存]   │      │ ┌──────────────────────────────────┐│
│   [重试]                       │      │ │ SyncStatusOverview（状态总览）   ││
│ ┌────────────────────────────┐│      │ │ [四张卡片横排]                   ││
│ │ Tab 内容面板                ││      │ └──────────────────────────────────┘│
│ │                             ││      │ [计划] [日志] [质量] [缓存] [重试] │
│ │                             ││ ──▶ │ ┌──────────────────────────────────┐│
│ └────────────────────────────┘│      │ │ Tab 内容面板                      ││
└───────────────────────────────┘      │ └──────────────────────────────────┘│
                                        └────────────────────────────────────┘
```

在 Tabs 上方插入状态总览卡片组，不影响现有 Tab 内容。

### 1.3 设计原则

1. **顶部概览**：在 Tabs 之上展示，始终可见
2. **按需隐藏**：可提供折叠/展开功能，默认展开
3. **权限统一**：复用现有 `SUPER_ADMIN` 权限守卫
4. **全量 POST**：使用 `apiClient.post()` 调用新端点

## 三、后端 API

### 3.1 新端点

| 功能         | 端点                                       | 请求体 | 说明               |
| ------------ | ------------------------------------------ | ------ | ------------------ |
| 同步状态总览 | `POST /tushare/admin/sync-status-overview` | `{}`   | 运维监控，聚合数据 |

### 3.2 响应类型定义

```typescript
/** 单个类别的同步状态汇总 */
export type SyncCategoryStatus = {
  category: TushareSyncCategory;
  totalTasks: number;
  successCount: number;
  failedCount: number;
  skippedCount: number;
  lastSyncAt: string | null; // 该类别最近一次同步时间
  consecutiveFailures: number; // 该类别下连续失败最多的任务数
};

/** 同步状态总览 */
export type SyncStatusOverview = {
  generatedAt: string; // 生成时间
  totalTasks: number; // 总任务数
  healthStatus: 'healthy' | 'degraded' | 'unhealthy'; // 整体健康状态
  categories: SyncCategoryStatus[]; // 按类别汇总
  lastFullSyncAt: string | null; // 最近一次全量同步时间
  lastIncrementalSyncAt: string | null; // 最近一次增量同步时间
  failedTaskNames: string[]; // 当前失败的任务名列表
};
```

> **注意**：以上是参照后端 sync-logs/summary + plans 的数据维度推测的响应结构。实际开发时以后端 Swagger / 接口文档为准，若响应字段不同需调整前端类型定义。

### 6.1 SyncStatusOverview

**Props**

```typescript
// 无 props — 组件内部自管数据
```

**内部状态**

```typescript
const [overview, setOverview] = useState<SyncStatusOverview | null>(null);
const [loading, setLoading] = useState(true);
const [collapsed, setCollapsed] = useState(false);
```

**数据加载**

```typescript
const fetchOverview = useCallback(async () => {
  setLoading(true);
  try {
    const data = await tushareSyncApi.getSyncStatusOverview();
    setOverview(data);
  } catch (err) {
    console.error('获取同步状态总览失败', err);
  } finally {
    setLoading(false);
  }
}, []);

useEffect(() => {
  fetchOverview();
}, [fetchOverview]);
```

**健康状态映射**

```typescript
const HEALTH_MAP = {
```

```tsx
                    mb: 1,
                  }}
                />

                <Typography variant="caption" color="text.secondary">

              </Paper>
            </Grid>

            {/* 2. 任务统计 */}
            <Grid size={{ xs: 6, md: 3 }}>
              <Paper variant="outlined" sx={{ p: 2, textAlign: 'center' }}>
                <Typography variant="h4">{overview.totalTasks}</Typography>

              </Paper>

              <Paper variant="outlined" sx={{ p: 2 }}>

                  最近同步
                </Typography>
                <Typography variant="body2">
                  增量：
                  {overview.lastIncrementalSyncAt ? fToNow(overview.lastIncrementalSyncAt) : '—'}
                </Typography>
                <Typography variant="body2">
                  全量：{overview.lastFullSyncAt ? fToNow(overview.lastFullSyncAt) : '—'}
                </Typography>
              </Paper>
            </Grid>

            {/* 4. 失败告警 */}
            <Grid size={{ xs: 6, md: 3 }}>
              <Paper

            </Grid>
          </Grid>

          <Typography variant="subtitle2" sx={{ mb: 1 }}>
```

## 九、错误处理与边界情况

| 场景               | 处理方式                                        |
| ------------------ | ----------------------------------------------- |
| 网络错误           | 静默失败（console.error），面板不渲染内容       |
| 接口返回空数据     | 显示默认空状态（totalTasks: 0，健康状态显示 —） |
| 非超级管理员       | 外层 TushareSyncView 权限守卫已拦截             |
| 响应结构与预期不符 | 实际开发时以后端接口文档调整类型定义            |
