## 一、模块定位与设计目标

### 1.1 为什么需要增强同步管理模块

当前同步管理页仅实现了同步计划展示和手动触发同步（2/11 接口，覆盖 18%），缺少数据运维必需的日志查看、质量监控、缓存统计和失败重试等功能。

| 痛点                             | 解决                                             |
| -------------------------------- | ------------------------------------------------ |
| 同步失败后无法查看日志定位原因   | 同步日志 Tab，支持按任务/状态/时间过滤分页查询   |
| 无法了解各任务最后同步状态       | 同步总览面板，展示各任务最后状态和连续失败次数   |
| 数据缺失无法感知                 | 数据质量 Tab，展示质量检查报告和缺失日期         |
| 缓存命中率无可视化               | 缓存统计 Tab，按命名空间展示命中率等指标         |
| 失败重试队列无法查看和干预       | 重试队列 Tab，查看队列状态并支持重置耗尽记录     |
| 无法手动触发数据质量检查         | 数据质量 Tab 提供手动触发按钮                    |

### 1.3 设计原则

1. **Tab 化设计**：在现有 `/tushare-sync` 路由下用 MUI Tabs 组织 5 个子面板，不新增路由
2. **保留现有功能**：同步计划和手动触发功能原封保留，作为第一个 Tab
3. **权限统一**：所有接口均要求 `SUPER_ADMIN` 角色，复用现有权限守卫
4. **全量 POST**：所有 API 均使用 `apiClient.post()`，查询参数通过 Body 传递
5. **日期格式化**：所有日期时间字段使用 `fDateTime()` 或 `fDate(value, 'YYYY-MM-DD')` 格式化

### 4.3 端点清单

| 功能               | 端点                                | 请求体                                                                                                     | 响应体                                                                               |
| ------------------ | ----------------------------------- | ---------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| 获取同步计划       | `POST /tushare/admin/plans`         | `{}`（空）                                                                                                 | `TushareSyncPlan[]`                                                                  |
| 手动触发同步       | `POST /tushare/admin/sync`          | `{ mode: 'incremental' \| 'full'; tasks?: string[] }`                                                      | `{ message: string }`（202 Accepted）                                                |
| 缓存统计           | `POST /tushare/admin/cache/stats`   | `{}`（空）                                                                                                 | `CacheMetricsData`                                                                   |
| 手动触发质量检查   | `POST /tushare/admin/quality/check` | `{}`（空）                                                                                                 | `{ message: string }`（202 Accepted）                                                |
| 质量检查报告       | `POST /tushare/admin/quality/report`| `{ days?: number }`                                                                                        | `DataQualityCheckItem[]`                                                             |
| 数据缺失日期       | `POST /tushare/admin/quality/gaps`  | `{ dataSet: string }`                                                                                      | `DataGapsResult`                                                                     |
| 数据校验异常日志   | `POST /tushare/admin/validation-logs`| `{ task?: string; limit?: number }`                                                                       | `ValidationLogItem[]`                                                                |
| 同步日志查询       | `POST /tushare/admin/sync-logs`     | `{ task?: string; status?: TushareSyncStatus; startDate?: string; endDate?: string; page?: number; pageSize?: number }` | `{ total: number; page: number; pageSize: number; items: SyncLogItem[] }`   |
| 同步状态汇总       | `POST /tushare/admin/sync-logs/summary` | `{}`（空）                                                                                             | `SyncLogSummaryItem[]`                                                               |
| 重试队列查询       | `POST /tushare/admin/retry-queue`   | `{ status?: TushareSyncRetryStatus; page?: number; pageSize?: number }`                                    | `{ total: number; page: number; pageSize: number; items: RetryQueueItem[] }`         |
| 重置重试队列       | `POST /tushare/admin/retry-queue/reset` | `{ task?: string }`                                                                                    | `{ message: string; count: number }`                                                 |

### 4.4 业务约束

| 约束                        | 规则                                                 |
| --------------------------- | ---------------------------------------------------- |
| 权限                        | 所有接口仅 `SUPER_ADMIN` 角色可访问                  |
| 同步日志分页                | `page` 默认 1，`pageSize` 默认 20，最大 100          |
| 重试队列分页                | `page` 默认 1，`pageSize` 默认 20，最大 100          |
| 质量报告查询范围            | `days` 默认 7，前端建议可选 7/14/30                  |
| 重置重试队列                | 仅重置 `EXHAUSTED` 状态记录为 `PENDING`              |
| 手动触发（同步/质量检查）   | 异步执行，立即返回 202 Accepted                      |
| 缓存统计                    | 实时查询，无分页                                     |
| 校验异常日志                | `limit` 默认 100，前端建议取最近 50~100 条            |
| 数据缺失日期                | 从最近一次 completeness fail/warn 记录中提取          |

## 九、数据加载策略

| 场景                             | 加载策略                                  | 加载时机          |
| -------------------------------- | ---------------------------------------- | ----------------- |
| 同步计划列表                     | 一次性加载全部                            | 同步计划 Tab 挂载 |
| 同步日志总览                     | 一次性加载全部任务汇总                    | 同步日志 Tab 挂载 |
| 同步日志列表                     | 分页加载（默认 20 条/页）                 | 同步日志 Tab 挂载 + 过滤/翻页 |
| 质量检查报告                     | 按天数加载（默认 7 天）                   | 数据质量 Tab 挂载 |
| 数据缺口                         | 按需查询                                  | 用户点击"查询缺失日期" |
| 校验异常日志                     | 一次性加载最近 100 条                     | 数据质量 Tab 挂载 |
| 缓存统计                         | 一次性加载                                | 缓存统计 Tab 挂载 |
| 重试队列                         | 分页加载（默认 20 条/页）                 | 重试队列 Tab 挂载 |

### 取消策略

所有 Tab 使用 `useEffect` + 清理函数标记 `cancelled`，Tab 切换时避免写入已卸载组件的 state。
