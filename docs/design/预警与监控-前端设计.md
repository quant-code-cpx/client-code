## 一、技术栈与约定

| 技术           | 版本/说明                                                                          |
| -------------- | ---------------------------------------------------------------------------------- |
| 框架           | React 19（Vite）                                                                   |
| UI 库          | MUI v7（`@mui/material`），使用 `sx` prop 做样式                                   |
| 图表           | ApexCharts（`react-apexcharts` / `apexcharts`）                                    |
| API 客户端     | 项目内 `src/api/client.ts` 的 `apiClient.get()` / `.post()`                        |
| 布局容器       | `DashboardContent`（来自 `src/layouts/dashboard`）                                 |
| 自定义图表封装 | `src/components/chart/` 下的 `Chart` 组件和 `useChart` hook                        |
| 路由           | 新增 `/alert` 及子路由                                                             |
| WebSocket      | `src/lib/socket.ts`，用于接收 `price-alert` / `market-anomaly-scan-completed` 事件 |

### 关键约定

1. **API 调用**：统一使用 `apiClient.get<T>(url, params)` 或 `apiClient.post<T>(url, body)`，不要用 fetch/axios。
2. **日期格式**：所有 `tradeDate` / `startDate` / `endDate` 参数使用 **YYYYMMDD**（8 位字符串）。
3. **错误处理**：组件内使用 `try/catch`，失败时在 UI 上显示提示（`Alert` 或 `Snackbar`），不能阻塞整页渲染。
4. **加载状态**：每个卡片/区域独立管理 `loading` 状态，使用 MUI `Skeleton` 占位。
5. **空数据保护**：使用数据前必须检查是否为 null / undefined / 空数组。
6. **颜色规则（涨跌）**：红涨 `error.main`、绿跌 `success.main`、中性 `text.secondary`。
7. **权限**：`scan` 类管理端点需 ADMIN 角色，前端需结合用户角色判断是否显示扫描按钮。

### 5.4 `AlertCalendarTimeline` — 事件时间线

**文件**：`src/sections/alert/alert-calendar-timeline.tsx`

- 将 events 按 `date` 字段分组（相同日期合并为一组）

```typescript
import { apiClient } from './client';

// ─── 类型定义 ───────────────────────────────────────────────

/** 事件日历查询 */
export type CalendarQuery = {
  startDate: string; // YYYYMMDD
  endDate: string; // YYYYMMDD
  tsCode?: string;
  types?: EventType[];
};

export type EventType = 'DISCLOSURE' | 'FLOAT' | 'DIVIDEND' | 'FORECAST';

export type CalendarEvent = {
  date: string; // YYYYMMDD
  tsCode: string;
  stockName: string;
  type: EventType;
  title: string;
  detail: string;
};

export type CalendarResponse = {
  startDate: string;
  endDate: string;
  totalCount: number;
  events: CalendarEvent[];
};

/** 价格预警规则 */
export type PriceAlertRuleType =
  | 'PCT_CHANGE_UP'
  | 'PCT_CHANGE_DOWN'
  | 'PRICE_ABOVE'
  | 'PRICE_BELOW'
  | 'LIMIT_UP'
  | 'LIMIT_DOWN';

export type PriceAlertRuleStatus = 'ACTIVE' | 'PAUSED' | 'DELETED';

export type PriceAlertRule = {
  id: string;
  userId: string;
  tsCode: string;
  ruleType: PriceAlertRuleType;
  threshold: number | null;
  memo: string | null;
  status: PriceAlertRuleStatus;
  triggerCount: number;
  lastTriggeredAt: string | null;
  createdAt: string;
  updatedAt: string;
};

export type CreatePriceRuleBody = {
  tsCode: string;
  ruleType: PriceAlertRuleType;
  threshold?: number;
  memo?: string;
};

export type UpdatePriceRuleBody = {
  tsCode?: string;
  ruleType?: PriceAlertRuleType;
  threshold?: number;
  memo?: string;
  status?: PriceAlertRuleStatus;
};

/** 异动监控 */
export type AnomalyType = 'VOLUME_SURGE' | 'CONSECUTIVE_LIMIT_UP' | 'LARGE_NET_INFLOW';

export type MarketAnomaly = {
  id: string;
  tradeDate: string; // YYYYMMDD
  tsCode: string;
  stockName: string;
  anomalyType: AnomalyType;
  value: number;
  threshold: number;
  detail: Record<string, unknown>;
  scannedAt: string;
};

export type AnomalyQuery = {
  tradeDate?: string; // YYYYMMDD
  type?: AnomalyType;
  tsCode?: string;
  page?: number;
  pageSize?: number;
};

export type AnomalyListResponse = {
  page: number;
  pageSize: number;
  total: number;
  items: MarketAnomaly[];
};

export type AnomalyScanResult = {
  tradeDate: string;
  volumeSurgeCount: number;
  limitUpCount: number;
  largeInflowCount: number;
  totalNew: number;
};

export type PriceRuleScanResult = {
  triggered: number;
};

// ─── API 函数 ────────────────────────────────────────────────

export const alertApi = {
  /** 事件日历查询 */
  getCalendar: (params: CalendarQuery) =>
    apiClient.get<CalendarResponse>('/api/alert/calendar', { params }),

  /** 获取当前用户所有价格预警规则 */
  getPriceRules: () => apiClient.get<PriceAlertRule[]>('/api/alert/price-rules'),

  /** 创建价格预警规则 */
  createPriceRule: (body: CreatePriceRuleBody) =>
    apiClient.post<PriceAlertRule>('/api/alert/price-rules', body),

  /** 更新价格预警规则 */
  updatePriceRule: (id: string, body: UpdatePriceRuleBody) =>
    apiClient.patch<PriceAlertRule>(`/api/alert/price-rules/${id}`, body),

  /** 删除价格预警规则（软删除） */
  deletePriceRule: (id: string) =>
    apiClient.delete<{ message: string }>(`/api/alert/price-rules/${id}`),

  /** 手动触发价格预警扫描（仅管理员） */
  scanPriceRules: () => apiClient.post<PriceRuleScanResult>('/api/alert/price-rules/scan'),

  /** 异动记录分页查询 */
  getAnomalies: (params: AnomalyQuery) =>
    apiClient.get<AnomalyListResponse>('/api/alert/anomalies', { params }),

  /** 手动触发异动扫描（仅管理员） */
  scanAnomalies: () => apiClient.post<AnomalyScanResult>('/api/alert/anomalies/scan'),
};
```

## 九、交互细节与状态流转

### 9.1 价格预警规则状态流转

```
新建 → ACTIVE
         ├─ 用户暂停 → PAUSED
         │              └─ 用户恢复 → ACTIVE
         └─ 用户删除 → DELETED（软删除，列表不再展示）
```

### 9.2 实时通知流

```
后端每日 19:00 自动扫描
       │
       ├─ 价格预警命中 → WebSocket `price-alert` → 特定用户
       │                 前端收到后：Snackbar 提示 + 刷新规则列表
       │
       └─ 异动扫描完成 → WebSocket `market-anomaly-scan-completed` → 所有用户
                         前端收到后：Snackbar 提示 + 刷新异动列表
```

### 9.3 错误场景处理

| 场景                    | 处理方式                                          |
| ----------------------- | ------------------------------------------------- |
| 日历日期范围 > 90 天    | 前端校验拦截，`Alert` 提示 `"范围不能超过 90 天"` |
| 创建规则时股票不存在    | 后端返回 400，前端 Snackbar 显示错误信息          |
| 删除他人规则（401/403） | 后端拒绝，前端 Snackbar 显示权限不足              |
| 扫描接口非 ADMIN 调用   | 按钮不渲染（前端隐藏），后端兜底 403              |
| 无交易日数据            | 异动页显示空状态 `"暂无异动数据"`                 |
