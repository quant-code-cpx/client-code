## 一、技术栈与现有架构评估

当前前端项目已具备实现本模块所需的基本能力：

| 能力             | 当前情况                                       | 结论      |
| ---------------- | ---------------------------------------------- | --------- |
| 页面分层         | `src/pages` + `src/sections`                   | ✅ 可复用 |
| API 客户端       | `src/api/client.ts`（统一 POST）               | ✅ 可复用 |
| 仪表盘布局       | `DashboardLayout`                              | ✅ 可复用 |
| 图表库           | `apexcharts` + `react-apexcharts`              | ✅ 可复用 |
| WebSocket 底座   | `src/lib/socket.ts` + `socket.io-client`       | ✅ 可复用 |
| 选股器 API       | `src/api/screener.ts`（含策略 CRUD + 过滤条件）| ✅ 可复用 |
| 股票搜索/列表    | `src/api/stock.ts`（含 StockListItem 类型）    | ✅ 可复用 |
| Markdown 渲染    | 需要新增 `react-markdown` 依赖                 | 🆕 新增   |
| 数字/时间格式化  | `fNumber` / `fPercent` / `fDate` / `fWanYuan`  | ✅ 可复用 |

### 关键设计决策

1. **不需要新引入 WebSocket 依赖**，继续复用 `src/lib/socket.ts`。
2. **所有 API 统一使用 POST**，ID 通过 Body 传递，不拼入 URL 路径。
3. **自选股和研究笔记整合为"研究工作台"**，使用 Tab 式布局，一个主入口 `/research`。
4. **条件订阅页面独立**，作为选股器功能的延伸，入口放在"股票"导航组下。
5. **策略草稿箱集成到回测工作台**，不独立成页，作为回测工作台的增强功能。
6. **需要新增 `react-markdown` + `remark-gfm`** 依赖用于研究笔记的 Markdown 渲染（编辑时使用纯 `<textarea>`，不引入重型富文本编辑器）。

### 3.3 交互流程

#### 自选组 CRUD

1. **页面加载** → 调用 `watchlistApi.getOverview()` 获取所有自选组概览卡片
2. **点击卡片** → 设置当前选中的自选组 ID，调用 `watchlistApi.getStocks(id)` 加载股票列表
3. **新建自选组** → 打开 `WatchlistCreateDialog`，填写名称/描述/是否默认 → 调用 `watchlistApi.create()`
4. **编辑自选组** → 打开 `WatchlistEditDialog`，预填现有数据 → 调用 `watchlistApi.update(id, data)`
5. **删除自选组** → 二次确认对话框 → 调用 `watchlistApi.delete(id)`
6. **拖拽排序** → 卡片可拖拽重排 → 调用 `watchlistApi.reorder(items)`

#### 股票管理

1. **添加单只** → 打开 `AddStockDialog`，包含股票搜索 Autocomplete + 备注 + 标签 + 目标价 → 调用 `watchlistApi.addStock(watchlistId, data)`
2. **批量导入** → 打开 `BatchImportDialog`，textarea 输入股票代码（逗号/换行分隔） → 解析后调用 `watchlistApi.batchAddStocks(watchlistId, stocks)`
3. **编辑股票** → 行内展开或弹出 Drawer → 编辑备注/标签/目标价 → 调用 `watchlistApi.updateStock(watchlistId, stockId, data)`
4. **移除股票** → 行操作按钮或批量选中后删除 → 调用 `watchlistApi.removeStock()` 或 `batchRemoveStocks()`
5. **查看详情** → 点击股票代码跳转到 `/stock/detail?code=<tsCode>`

#### 行情展示

- 股票列表表格的 `quote` 数据由后端 `getStocks` 接口一并返回（后端已做 JOIN 查询）
- 涨跌幅用红绿色标识（`pctChg > 0` 红色，`< 0` 绿色）
- 目标价与现价对比：达到目标价时高亮提示

### 4.4 交互流程

#### 笔记列表

1. **页面加载** → 调用 `researchNoteApi.list({})` 获取笔记列表 + 调用 `researchNoteApi.getTags()` 获取标签列表
2. **筛选** → 选择标签/输入股票代码/关键词 → 重新调用 `researchNoteApi.list(filters)` 刷新
3. **排序切换** → 更新时间 / 创建时间 → 作为参数传入 API
4. **点击笔记卡片** → 跳转到 `/research/notes/:noteId`
5. **新建笔记** → 跳转到 `/research/notes/new`

#### 笔记详情/编辑

1. **加载**：`noteId !== 'new'` → 调用 `researchNoteApi.getById(noteId)` 获取详情
2. **编辑模式切换**：默认预览模式，点击编辑切换为左右分栏编辑模式
3. **保存**：
   - 新建：调用 `researchNoteApi.create(data)`
   - 更新：调用 `researchNoteApi.update(noteId, data)`
4. **删除**：二次确认 → 调用 `researchNoteApi.delete(noteId)` → 跳回列表
5. **置顶/取消置顶**：调用 `researchNoteApi.update(noteId, { isPinned: !current })`
6. **关联股票**：Autocomplete 搜索选择股票代码，可选关联

#### 标签管理

- 新建笔记时可直接输入新标签（Chip 输入模式）
- 标签筛选下拉从 `getTags()` 获取用户已用标签
- 标签采用 AND 语义筛选（与后端一致）

### 4.5 Markdown 渲染方案

新增依赖：`react-markdown` + `remark-gfm`（支持表格/删除线/任务列表等 GFM 扩展）

```tsx
import ReactMarkdown from 'react-markdown'
import remarkGfm from 'remark-gfm'

<ReactMarkdown remarkPlugins={[remarkGfm]}>
  {content}
</ReactMarkdown>
```

样式与 MUI Typography 集成，在渲染时通过 `components` prop 将标题映射到 `<Typography variant="h4">` 等。

### 6.3 交互流程

1. **加载草稿** → 调用 `draftApi.getDrafts()` 获取列表 → 展示在 Drawer 中
2. **选择草稿** → 调用 `draftApi.getById(id)` 获取详细 config → 回填到表单各字段
3. **保存草稿** → 手动点击"保存当前为新草稿"或自动保存：
   - 新建：调用 `draftApi.create({ name, config })`
   - 更新：调用 `draftApi.update(id, { config })`
4. **从草稿提交回测** → 先加载草稿到表单 → 用户点"开始回测"正常提交（或在 Drawer 中点"直接提交"调用 `draftApi.submit(id)`）
5. **删除草稿** → 二次确认 → 调用 `draftApi.delete(id)`

### 6.4 自动保存机制

```typescript
// 使用 useRef + setTimeout 实现防抖自动保存
const autoSaveTimer = useRef<NodeJS.Timeout>()
const currentDraftId = useRef<number | null>(null)

const handleFormChange = (config: Record<string, unknown>) => {
  if (autoSaveTimer.current) clearTimeout(autoSaveTimer.current)
  autoSaveTimer.current = setTimeout(async () => {
    if (currentDraftId.current) {
      await draftApi.update(currentDraftId.current, { config })
      // 显示"已自动保存"提示
    }
  }, 5000) // 5 秒防抖
}
```

### 8.1 自选股实时更新

在 `watchlist-view.tsx` 中通过 WebSocket 订阅自选组变更：

```typescript
import { getSocket } from 'src/lib/socket'

useEffect(() => {
  if (!selectedId) return

  const socket = getSocket()
  socket.emit('subscribe_watchlist', { watchlistId: selectedId })

  const handleUpdate = (data: { watchlistId: number; action: string; tsCode?: string }) => {
    if (data.watchlistId === selectedId) {
      // 刷新股票列表
      refreshStocks()
    }
  }

  socket.on('watchlist_updated', handleUpdate)

  return () => {
    socket.emit('unsubscribe_watchlist', { watchlistId: selectedId })
    socket.off('watchlist_updated', handleUpdate)
  }
}, [selectedId])
```

// src/sections/screener-subscription/hooks/use-subscription-alerts.ts

export function useSubscriptionAlerts() {
  const router = useRouter()

  useEffect(() => {
    const socket = getSocket()

    const handleAlert = (data: {
      subscriptionId: number
      subscriptionName: string
      tradeDate: string
      newEntryCodes: string[]
      exitCodes: string[]
      totalMatch: number
    }) => {
      // 使用 Snackbar 展示通知
      enqueueSnackbar(
        `「${data.subscriptionName}」发现 ${data.newEntryCodes.length} 只新股票`,
        {
          variant: 'info',
          action: () => (
            <Button onClick={() => router.push(`/stock/subscription/${data.subscriptionId}`)}>
              查看详情
            </Button>
          ),
        }
      )
    }

    socket.on('screener_subscription_alert', handleAlert)
    return () => { socket.off('screener_subscription_alert', handleAlert) }
  }, [router])
}
