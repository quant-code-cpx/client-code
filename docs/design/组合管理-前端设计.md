## 一、模块定位与设计目标

### 1.1 为什么需要组合管理模块

组合管理是量化投资闭环的核心环节，连接策略研究与实际投资决策。当前前端已有回测工作台和策略管理，但缺乏：

| 痛点                         | 解决                                                   |
| ---------------------------- | ------------------------------------------------------ |
| 无法将回测结果转化为可跟踪组合 | 创建虚拟组合，导入回测持仓或手动建仓                   |
| 无法跟踪组合实时盈亏          | 当日浮动盈亏 + 历史净值曲线                             |
| 无法分析组合风险敞口          | 行业分布、仓位集中度、市值分布、Beta 系数四维风险分析   |
| 缺乏风控预警机制              | 风控规则配置 + 自动检测 + 历史违规记录                   |
| 持仓调整无记录                | 完整的持仓管理（添加/更新/删除），成本自动计算           |

### 1.2 与其他模块的关系

```text
回测工作台                          组合管理                         策略管理
┌──────────────┐                 ┌──────────────┐                ┌──────────────┐
│ 回测详情页    │── 导入持仓 ──▶│ 创建组合      │                │ 策略管理      │
│ 最终持仓列表  │                │ 组合列表/详情 │                │ 运行回测 ─┐  │
└──────────────┘                │ 持仓管理      │                └───────────│──┘
                                │ 盈亏追踪      │                            │
                                │ 风险分析      │                            ▼
首页仪表盘                       │ 风控规则      │                ┌──────────────┐
┌──────────────┐                └──────────────┘                │ 回测任务历史  │
│ 组合收益概览  │◀── 摘要数据 ──┘                                └──────────────┘
└──────────────┘
```

- 未来首页仪表盘可展示组合收益概览卡片
- 回测详情页未来可增加 "创建模拟组合" 按钮（将回测最终持仓导入）
- 组合详情页可链接到对应股票详情页

### 1.3 设计原则

1. **Tab 式详情页**：组合详情使用 Tab 布局（持仓 / 盈亏 / 风险 / 风控），避免单页过长
2. **鼠标友好**：所有配置使用结构化表单（Select / TextField / Slider / Switch），禁止原始 JSON
3. **实时反馈**：盈亏和风险数据带刷新按钮，支持手动刷新（后端缓存 5~10 分钟）
4. **渐进增强**：先实现组合 CRUD + 持仓管理，再叠加盈亏/风险/风控功能
5. **空值安全**：所有数值字段可能为 `null`（非交易日、数据缺失），渲染时需做兜底

## 二、技术栈与复用分析

当前前端项目已具备实现本模块所需的全部基本能力：

| 能力           | 当前状况                                           | 结论      |
| -------------- | -------------------------------------------------- | --------- |
| 页面分层       | `src/pages` + `src/sections`                       | ✅ 可复用 |
| API 客户端     | `src/api/client.ts`（统一 POST）                   | ✅ 可复用 |
| 仪表盘布局     | `DashboardLayout` + `DashboardContent`             | ✅ 可复用 |
| 图表库         | `apexcharts` + `react-apexcharts`                  | ✅ 可复用 |
| 数字格式化     | `fNumber` / `fPercent` / `fCurrency` / `fWanYuan`  | ✅ 可复用 |
| 时间格式化     | `fDate` / `fDateTime` / `fToNow`                   | ✅ 可复用 |
| 表格组件       | MUI `Table` / `TableContainer` / `TablePagination` | ✅ 可复用 |
| 股票搜索       | `useStockSearch()` hook                            | ✅ 可复用 |
| Label 组件     | `src/components/label`（状态标签）                 | ✅ 可复用 |
| Scrollbar      | `src/components/scrollbar`                         | ✅ 可复用 |

### 关键设计决策

1. **不需要新引入第三方依赖**，全部使用现有技术栈
2. **所有 API 统一使用 POST**，ID 通过 Body 传递
3. **组合详情页使用 Tab 布局**：持仓管理 / 盈亏分析 / 风险分析 / 风控规则
4. **图表使用 ApexCharts**：净值曲线（area chart）、行业分布（pie chart）、市值分布（bar chart）
5. **金额格式化**：使用 `fCurrency` 或 `fWanYuan` 显示金额，`fPercent` 显示百分比

## 四、后端 API 映射清单

### 4.1 端点总览（20 个端点，全部 POST）

#### 组合 CRUD（5 端点）

| 功能     | 端点                    | 请求体                                                     | 响应体                |
| -------- | ----------------------- | ---------------------------------------------------------- | --------------------- |
| 创建组合 | `POST /portfolio/create` | `{ name: string; description?: string; initialCash: number }` | `PortfolioCreatedDto` |
| 组合列表 | `POST /portfolio/list`   | `{}`（无参数）                                              | `PortfolioListItemDto[]` |
| 组合详情 | `POST /portfolio/detail` | `{ portfolioId: string }`                                   | `PortfolioDetailDto`  |
| 更新组合 | `POST /portfolio/update` | `{ id: string; name?: string; description?: string }`       | `PortfolioUpdatedDto` |
| 删除组合 | `POST /portfolio/delete` | `{ portfolioId: string }`                                   | `SuccessDto`          |

#### 持仓管理（3 端点）

| 功能     | 端点                         | 请求体                                                                              | 响应体          |
| -------- | ---------------------------- | ----------------------------------------------------------------------------------- | --------------- |
| 添加持仓 | `POST /portfolio/holding/add`    | `{ portfolioId: string; tsCode: string; quantity: number; avgCost: number }` | `HoldingItemDto` |
| 更新持仓 | `POST /portfolio/holding/update` | `{ holdingId: string; quantity: number; avgCost: number }`                   | `HoldingItemDto` |
| 删除持仓 | `POST /portfolio/holding/remove` | `{ holdingId: string }`                                                      | `SuccessDto`     |

#### 盈亏分析（2 端点）

| 功能         | 端点                        | 请求体                                                              | 响应体                 |
| ------------ | --------------------------- | ------------------------------------------------------------------- | ---------------------- |
| 当日浮动盈亏 | `POST /portfolio/pnl/today`   | `{ portfolioId: string }`                                           | `PnlTodayDto`          |
| 历史净值曲线 | `POST /portfolio/pnl/history` | `{ portfolioId: string; startDate: string; endDate: string }` | `PnlHistoryItemDto[]`  |

> `startDate` / `endDate` 格式为 `YYYYMMDD`（8 位纯数字字符串）

#### 风险分析（4 端点）

| 功能         | 端点                            | 请求体                     | 响应体                      |
| ------------ | ------------------------------- | -------------------------- | --------------------------- |
| 行业分布     | `POST /portfolio/risk/industry`   | `{ portfolioId: string }`  | `IndustryDistributionDto`   |
| 仓位集中度   | `POST /portfolio/risk/position`   | `{ portfolioId: string }`  | `PositionConcentrationDto`  |
| 市值分布     | `POST /portfolio/risk/market-cap` | `{ portfolioId: string }`  | `MarketCapDistributionDto`  |
| Beta 系数    | `POST /portfolio/risk/beta`       | `{ portfolioId: string }`  | `BetaAnalysisDto`           |

#### 风控规则管理（4 端点）

| 功能         | 端点                          | 请求体                                                                                            | 响应体        |
| ------------ | ----------------------------- | ------------------------------------------------------------------------------------------------- | ------------- |
| 规则列表     | `POST /portfolio/rule/list`   | `{ portfolioId: string }`                                                                         | `RiskRuleDto[]` |
| 创建/更新规则 | `POST /portfolio/rule/upsert` | `{ portfolioId: string; ruleType: string; threshold: number; isEnabled?: boolean; memo?: string }` | `RiskRuleDto`  |
| 修改规则     | `POST /portfolio/rule/update` | `{ ruleId: string; threshold: number; isEnabled: boolean; memo?: string }`                         | `RiskRuleDto`  |
| 删除规则     | `POST /portfolio/rule/delete` | `{ ruleId: string }`                                                                               | `SuccessDto`   |

#### 风控检测（2 端点）

| 功能         | 端点                              | 请求体                                       | 响应体                 |
| ------------ | --------------------------------- | -------------------------------------------- | ---------------------- |
| 执行风控检测 | `POST /portfolio/risk/check`      | `{ portfolioId: string }`                    | `RiskCheckResultDto`   |
| 历史违规记录 | `POST /portfolio/risk/violations` | `{ portfolioId: string; limit?: number }`    | `ViolationRecordDto[]` |

### 4.4 业务约束

| 约束             | 规则                                   |
| ---------------- | -------------------------------------- |
| 组合名称长度     | 1~100 字符                             |

### 4.5 后端缓存策略

| 资源             | 缓存时间 | 说明                           |
| ---------------- | -------- | ------------------------------ |
| 组合详情         | 5 分钟   | 含持仓估值，依赖实时行情       |
| 当日浮动盈亏     | 5 分钟   | 交易时段内需手动刷新           |
| 历史净值曲线     | 1 小时   | 历史数据相对稳定               |
| 行业分布         | 10 分钟  | 分布数据变化不频繁             |
| 仓位集中度       | 10 分钟  | 同上                           |
| 市值分布         | 10 分钟  | 同上                           |
| Beta 系数        | 1 小时   | 依赖历史数据计算               |

> 前端在对应数据区域提供"刷新"按钮，用户可手动触发最新数据获取。

## 九、数据加载策略

| 场景                 | 策略                                                     |
| -------------------- | -------------------------------------------------------- |
| 组合列表初始加载     | 进入页面即请求 `listPortfolios()`，无分页                |
| 组合详情初始加载     | 进入详情页即请求 `getPortfolioDetail()`                  |
| 持仓管理 Tab         | 数据来自详情接口，无需额外请求                           |
| 盈亏分析 Tab         | 切换到该 Tab 时请求 `getPnlToday()` + `getPnlHistory()` |
| 风险分析 Tab         | 切换到该 Tab 时并行请求四个风险分析接口                  |
| 风控规则 Tab         | 切换到该 Tab 时请求 `listRiskRules()` + `getViolations()` |
| 创建/更新/删除后     | 操作成功后刷新对应数据（列表或详情）                     |
| 手动刷新             | 各 Tab 提供刷新按钮，重新请求对应数据                    |
| 风控检测             | 用户主动触发，请求后展示结果                             |

### 懒加载 Tab 数据

详情页使用 Tab 布局，每个 Tab 的数据在首次切换时加载，使用 ref 标记是否已加载过：

```typescript
const loadedTabs = useRef<Set<string>>(new Set());

useEffect(() => {
  if (loadedTabs.current.has(currentTab)) return;
  loadedTabs.current.add(currentTab);

  switch (currentTab) {
    case 'pnl':
      loadPnlData();
      break;
    case 'risk':
      loadRiskData();
      break;
    case 'rules':
      loadRuleData();
      break;
    default:
      break;
  }
}, [currentTab]);
```
