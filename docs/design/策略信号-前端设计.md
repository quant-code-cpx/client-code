## 一、模块定位与设计目标

### 1.1 为什么需要策略信号模块

策略信号是从"策略回测"到"实盘跟踪"的桥梁。当前用户完成策略回测后，只能看到历史模拟结果，无法获取每日最新的交易信号。Signal 模块解决以下痛点：

| 痛点                             | 解决方案                                     |
| -------------------------------- | -------------------------------------------- |
| 策略回测结果只是历史模拟         | 激活信号后，每日自动生成最新交易信号         |
| 不知道策略当前建议买什么、卖什么 | 信号面板展示 BUY/SELL/HOLD 明细和目标权重    |
| 无法将策略信号与组合持仓对比     | 支持关联组合，激活策略漂移检测               |
| 多个策略的信号分散，缺乏统一入口 | 独立信号页面汇总所有已激活策略的最新信号     |
| 无法回顾历史信号                 | 信号历史页按交易日分组展示，支持日期范围筛选 |

### 1.2 与策略管理 / 组合管理的关系

```
策略管理                   Signal 模块                   组合管理
┌──────────────┐          ┌──────────────┐            ┌──────────────┐
│ 策略详情页    │── 激活 ──▶│ SignalActivation│── 关联 ──▶│ 组合详情页    │
│ 信号管理卡片  │  信号生成  │ 每日信号列表   │  组合 ID   │ 漂移检测      │
└──────────────┘          └──────────────┘            └──────────────┘
       │                         │
       │ strategies/activate     │ signal/latest
       │ strategies/deactivate   │ signal/history
       │ strategies/list         │
       ▼                         ▼
  POST /signal/*            POST /signal/*
```

- **激活入口**：在策略详情页新增「信号管理」卡片，用户可在此激活/停用信号生成
- **信号查看**：独立的信号页面展示所有激活策略的最新信号和历史记录
- **组合联动**：激活时可选关联组合 ID，后端自动在生成信号时执行漂移检测

### 1.3 设计原则

1. **双入口**：策略详情页提供激活/停用操作；独立信号页提供统一的信号查看
2. **信息层级清晰**：先看策略维度汇总，再看个股信号明细
3. **涨跌配色**：BUY 用 `success.main`（绿）、SELL 用 `error.main`（红）、HOLD 用 `text.secondary`
4. **空状态友好**：未激活策略时，引导用户前往策略管理页激活

## 四、API 层设计

### 4.1 新增 `src/api/signal.ts`

```typescript
import type { Signal } from '@abort-controller/abort-controller';

import { apiClient } from './client';

// ── Types ─────────────────────────────────────────────────────

/** 激活信号请求 */
export interface ActivateSignalParams {
  strategyId: string;
  portfolioId?: string;
  universe?: string;
  benchmarkTsCode?: string;
  lookbackDays?: number;
  alertThreshold?: number;
}

/** 停用信号请求 */
export interface DeactivateSignalParams {
  strategyId: string;
}

/** 信号激活项 */
export interface SignalActivationItem {
  id: string;
  strategyId: string;
  strategyName: string;
  portfolioId: string | null;
  isActive: boolean;
  universe: string;
  benchmarkTsCode: string;
  lookbackDays: number;
  alertThreshold: number;
  lastSignalDate: string | null;
  createdAt: string;
  updatedAt: string;
}

/** 交易信号条目 */
export interface TradingSignalItem {
  tsCode: string;
  stockName: string;
  action: 'BUY' | 'SELL' | 'HOLD';
  targetWeight: number | null;
  confidence: number | null;
}

/** 最新信号查询参数 */
export interface LatestSignalQuery {
  strategyId?: string;
  tradeDate?: string;
}

/** 最新信号响应 */
export interface LatestSignalResponse {
  strategyId: string;
  strategyName: string;
  tradeDate: string;
  signals: TradingSignalItem[];
  generatedAt: string;
}

/** 信号历史查询参数 */
export interface SignalHistoryQuery {
  strategyId: string;
  startDate?: string;
  endDate?: string;
  page?: number;
  pageSize?: number;
}

/** 信号历史分组 */
export interface SignalHistoryGroup {
  tradeDate: string;
  signalCount: number;
  signals: TradingSignalItem[];
}

/** 信号历史响应 */
export interface SignalHistoryResponse {
  strategyId: string;
  total: number;
  page: number;
  pageSize: number;
  groups: SignalHistoryGroup[];
}

// ── API Functions ─────────────────────────────────────────────

export const signalApi = {
  /** 激活策略的每日信号生成 */
  activate: (params: ActivateSignalParams): Promise<SignalActivationItem> =>
    apiClient.post<SignalActivationItem>('/api/signal/strategies/activate', params),

  /** 停用策略信号生成 */
  deactivate: (params: DeactivateSignalParams): Promise<SignalActivationItem> =>
    apiClient.post<SignalActivationItem>('/api/signal/strategies/deactivate', params),

  /** 查询已激活策略列表 */
  listActivations: (): Promise<SignalActivationItem[]> =>
    apiClient.post<SignalActivationItem[]>('/api/signal/strategies/list', {}),

  /** 查询最新信号 */
  getLatestSignals: (params: LatestSignalQuery): Promise<LatestSignalResponse[]> =>
    apiClient.post<LatestSignalResponse[]>('/api/signal/latest', params),

  /** 查询信号历史 */
  getSignalHistory: (params: SignalHistoryQuery): Promise<SignalHistoryResponse> =>
    apiClient.post<SignalHistoryResponse>('/api/signal/history', params),
};
```

## 八、交互说明

### 8.1 信号操作颜色规范

| 操作 | Label 颜色 | 行背景色（浅）                                           |
| ---- | ---------- | -------------------------------------------------------- |
| BUY  | `success`  | `varAlpha(theme.vars.palette.success.mainChannel, 0.08)` |
| SELL | `error`    | `varAlpha(theme.vars.palette.error.mainChannel, 0.08)`   |
| HOLD | `default`  | 无背景或 `grey.50`                                       |

### 8.2 Loading 状态

- 每个 API 请求独立管理 `loading` 状态
- 概览卡片 loading 时使用 `Skeleton` 占位（3 个 Skeleton 卡片）
- 信号明细 loading 时使用表格 `Skeleton` 行占位

### 8.3 错误处理

- API 失败时显示 inline `Alert` 组件提示错误信息
- 不阻塞整页渲染，各区域独立容错

### 8.4 置信度显示

- `confidence` 为 `null` 时显示 `—`
- 有值时显示百分比格式（如 `82%`）并用进度条辅助可视化
- 0.8+ 绿色、0.5~0.8 橙色、<0.5 红色

### 8.5 目标权重显示

- `targetWeight` 为 `null` 时显示 `—`
- 有值时显示百分比格式（如 `8.5%`），保留一位小数

### 8.6 权限说明

- 信号管理为用户级功能，无管理员专用端点
- 所有端点均需 JWT 认证，前端无需额外权限判断

## 九、数据类型定义速查

所有类型定义在 `src/api/signal.ts` 中（详见第四节），核心类型：

| 类型                    | 用途                               |
| ----------------------- | ---------------------------------- |
| `SignalActivationItem`  | 激活策略项（列表/卡片/激活响应）   |

| `LatestSignalResponse`  | 最新信号响应（含策略名+信号数组）  |
| `SignalHistoryResponse` | 信号历史响应（含分组+分页信息）    |
| `SignalHistoryGroup`    | 信号历史分组（单交易日的信号集合） |
