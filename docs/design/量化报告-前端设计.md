## 一、模块定位与设计目标

### 1.1 为什么需要量化报告模块

量化研究产出需要结构化的报告来沉淀和分享。当前前端已有回测、个股分析、组合管理、策略管理，但缺乏统一的报告生成与管理能力：

| 痛点                           | 解决                                                         |
| ------------------------------ | ------------------------------------------------------------ |
| 回测结果分散，难以形成完整报告 | 一键生成回测报告（策略信息、收益指标、净值曲线、交易明细）   |
| 个股研究结论无法输出为独立文档 | 生成个股研报（概览、行情历史、财务摘要、十大股东、分红记录） |
| 组合持仓分析无法导出分享       | 生成组合报告（概览、持仓明细、行业分布）                     |
| 策略研究缺乏综合分析报告       | 生成策略研究报告（回测表现、持仓分析、风险评估、交易日志）   |
| 已生成报告无法统一管理和检索   | 报告列表页：筛选、搜索、查看、下载、删除                     |
| 报告格式单一                   | 支持 JSON（数据查看）、HTML（网页预览）、PDF（下载分享）     |

### 1.3 设计原则

1. **鼠标友好**：报告类型、输出格式等使用 Select / ToggleButtonGroup / Switch，禁止手动输入
2. **异步感知**：报告生成是后台异步任务，前端需轮询状态，提供明确的进度反馈
3. **多格式支持**：JSON 在页面内渲染结构化数据，HTML/PDF 提供下载链接
4. **类型化查看器**：4 种报告类型各有专属查看器组件，按 `Report.data` 结构渲染
5. **渐进增强**：先实现报告列表 + 详情查看，再逐步在其他页面挂载"生成报告"入口

## 二、技术栈与复用分析

| 能力         | 当前状况                                           | 结论      |
| ------------ | -------------------------------------------------- | --------- |
| 页面分层     | `src/pages` + `src/sections`                       | ✅ 可复用 |
| API 客户端   | `src/api/client.ts`（统一 POST）                   | ✅ 可复用 |
| 仪表盘布局   | `DashboardLayout` + `DashboardContent`             | ✅ 可复用 |
| 图表库       | `apexcharts` + `react-apexcharts`                  | ✅ 可复用 |
| Label 组件   | `src/components/label`（状态标签）                 | ✅ 可复用 |
| Scrollbar    | `src/components/scrollbar`                         | ✅ 可复用 |
| Iconify 图标 | `src/components/iconify`                           | ✅ 可复用 |
| 数字格式化   | `fNumber` / `fPercent` / `fCurrency` / `fWanYuan`  | ✅ 可复用 |
| 时间格式化   | `fDate` / `fDateTime`                              | ✅ 可复用 |
| 表格组件     | MUI `Table` / `TableContainer` / `TablePagination` | ✅ 可复用 |

### 关键设计决策

1. **不需要新引入第三方依赖**，全部使用现有技术栈
2. **所有 API 统一使用 POST**，ID 通过 Body 传递
3. **报告详情页根据 `type` 字段动态加载对应查看器组件**
4. **图表使用 ApexCharts**：净值曲线（area）、回撤曲线（area）、月度收益（heatmap/bar）、行业分布（pie）
5. **JSON 格式报告**：`data` 字段直接在页面渲染结构化内容
6. **HTML/PDF 格式报告**：通过 `filePath` 提供下载/预览链接
7. **异步状态轮询**：PENDING/GENERATING 状态使用 `setInterval` 每 3 秒重新请求详情

## 四、后端 API 映射清单

### 4.1 端点总览（7 个端点，全部 POST）

#### 报告生成（4 端点）

| 功能             | 端点                             | 请求体                            | 响应体   |
| ---------------- | -------------------------------- | --------------------------------- | -------- |
| 生成回测报告     | `POST /report/backtest`          | `CreateBacktestReportDto`         | `Report` |
| 生成个股研报     | `POST /report/stock`             | `CreateStockReportDto`            | `Report` |
| 生成组合报告     | `POST /report/portfolio`         | `CreatePortfolioReportDto`        | `Report` |
| 生成策略研究报告 | `POST /report/strategy-research` | `CreateStrategyResearchReportDto` | `Report` |

#### 报告管理（3 端点）

| 功能     | 端点                  | 请求体                        | 响应体                             |
| -------- | --------------------- | ----------------------------- | ---------------------------------- |
| 报告列表 | `POST /report/list`   | `{ type?, page?, pageSize? }` | `{ items, total, page, pageSize }` |
| 报告详情 | `POST /report/detail` | `{ reportId: string }`        | `Report`（含 `data` 字段）         |
| 删除报告 | `POST /report/delete` | `{ reportId: string }`        | `{ deleted: true }`                |

### 4.2 请求 DTO 定义

#### CreateBacktestReportDto

```typescript
{
  runId: string;            // 回测运行 ID
  title?: string;           // 报告标题（可选，后端自动生成默认标题）

}
```

#### CreateStockReportDto

```typescript
{
  tsCode: string;           // 股票代码（如 "000001.SZ"）
  title?: string;
  format?: ReportFormat;
}
```

#### CreatePortfolioReportDto

```typescript
{
  portfolioId: string;      // 组合 ID
  title?: string;
  format?: ReportFormat;
}
```

#### CreateStrategyResearchReportDto

```typescript
{
  backtestRunId: string;    // 必填，关联的回测运行 ID
  strategyId?: string;      // 可选，关联的策略 ID
  portfolioId?: string;     // 可选，关联的组合 ID
  title?: string;
  format?: ReportFormat;
  sections?: {              // 可选章节控制
    performance?: boolean;  // 回测表现，默认 true
    holdings?: boolean;     // 持仓分析，默认 true
```

### 4.3 响应 DTO 定义

```typescript

  type: ReportType; // 报告类型
  title: string; // 报告标题
  params: Record<string, unknown>; // 生成参数快照
  data: Record<string, unknown> | null; // 结构化报告数据（JSON 格式时有值）

  fileSize: number | null; // 文件大小（字节）
  createdAt: string; // 创建时间（ISO DateTime）
```

### 4.6 业务约束

| 约束                 | 说明                                                      |
| -------------------- | --------------------------------------------------------- |
| 报告 ID 格式         | `string`（cuid），非自增整数                              |
| 异步生成             | 生成请求立即返回 `Report`（status=PENDING），后台异步执行 |
| 状态流转             | PENDING → GENERATING → COMPLETED \| FAILED                |
| JSON 格式            | `data` 字段有值，`filePath` 为 null                       |
| HTML/PDF 格式        | `filePath` 有值，`data` 可能为 null                       |
| 策略研究报告章节控制 | `sections` 各字段默认 true，可手动关闭                    |
| 权限                 | 用户只能查看/删除自己的报告，JWT 自动过滤                 |

## 九、数据加载策略

| 场景              | 策略                                                     |
| ----------------- | -------------------------------------------------------- |
| 报告列表          | 筛选/翻页时调用 `listReports`，不缓存                    |
| 报告详情          | 进入页面调用 `getReportDetail`，完成后停止               |
| 异步状态轮询      | status 为 PENDING/GENERATING 时 `setInterval(3000)` 轮询 |
| 轮询终止          | status 变为 COMPLETED/FAILED 时 `clearInterval`          |
| 组件卸载清理      | `useEffect` return 中 `clearInterval` 防止内存泄漏       |
| 查看器数据        | 从 `report.data` 直接传入，无需额外请求                  |
| 交易记录/月度收益 | 数据量可能较大，使用客户端分页（前端 slice）             |

### 轮询代码示例

```typescript
useEffect(() => {
  if (!report || (report.status !== 'PENDING' && report.status !== 'GENERATING')) return;

  const timer = setInterval(async () => {
    const updated = await reportApi.getReportDetail({ reportId: report.id });
    setReport(updated);
    if (updated.status === 'COMPLETED' || updated.status === 'FAILED') {
      clearInterval(timer);
      setPolling(false);
    }
  }, 3000);

  setPolling(true);

  return () => clearInterval(timer);
}, [report?.id, report?.status]);
```

## 十一、交互要求与 UX 细节

### 11.1 操作反馈

| 操作     | 反馈                                            |
| -------- | ----------------------------------------------- |
| 提交生成 | Snackbar success："报告生成任务已提交"          |
| 删除报告 | 确认弹窗 → Snackbar success："报告已删除"       |
| 生成失败 | 详情页 Alert severity="error" 显示 errorMessage |
| 生成完成 | 轮询自动刷新，状态 Label 变为 "已完成"          |
| API 错误 | Snackbar error 显示错误信息                     |

### 11.2 加载状态

| 场景         | 处理方式                                                  |
| ------------ | --------------------------------------------------------- |
| 列表加载中   | 表格区域显示 `Skeleton`（5 行）                           |
| 详情加载中   | 全页 `CircularProgress` 居中                              |
| 报告生成中   | 详情页显示 `CircularProgress` + "报告正在生成中，请稍候…" |
| 报告数据为空 | `data` 为 null 且 status=COMPLETED → "报告数据为空" 提示  |
| 列表为空     | 空状态插图 + "暂无报告，点击"生成报告"开始"               |

### 11.3 空值处理

- 所有 `number | null` 字段：null → 显示 `'-'`
- 所有 `string | null` 字段：null → 显示 `'-'`
- `fileSize` 为 null → 显示 `'-'`
- `completedAt` 为 null → 显示 `'-'`

### 11.4 鼠标友好控件

| 字段         | 控件                | 说明                    |
| ------------ | ------------------- | ----------------------- |
