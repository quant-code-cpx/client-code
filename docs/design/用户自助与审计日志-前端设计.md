## 一、模块定位与设计目标

### 1.1 为什么需要用户自助与审计日志功能

当前用户模块已实现管理员级别的 CRUD（8/11 接口，覆盖 73%），但缺少 3 个端点的前端对接：

| 痛点                               | 解决                                                     |
| ---------------------------------- | -------------------------------------------------------- |
| 用户无法自助修改密码               | 个人资料页提供修改密码表单，验证旧密码后设置新密码       |
| 用户无法自助修改昵称、邮箱、微信号 | 个人资料页提供编辑资料表单，修改后即时刷新全局 Profile    |
| 管理员无法查看用户操作审计记录     | 用户管理页新增"审计日志"Tab，支持按操作者/目标/类型/时间筛选 |

- **个人资料页**：新增独立路由 `/profile`，所有登录用户可访问
- **审计日志**：嵌入现有 `/user-manage` 页面作为第二个 Tab，仅 `ADMIN` 及以上角色可见

### 1.3 设计原则

1. **最小改动**：个人资料为新增独立页面，审计日志以 Tab 形式嵌入现有用户管理页，不破坏已有功能
2. **即时反馈**：修改资料后立即调用 `loadProfile()` 刷新全局用户状态，头像下拉菜单等自动更新
3. **全量 POST**：所有 API 均使用 `apiClient.post()`，查询参数通过 Body 传递
4. **日期格式化**：所有日期时间字段使用 `fDateTime()` 格式化
5. **权限分离**：个人资料页任何登录用户可访问；审计日志仅 `ADMIN` 及以上角色可访问

### 4.3 端点清单

| 功能         | 端点                             | 请求体                | 响应体              | 权限要求       |
| ------------ | -------------------------------- | --------------------- | ------------------- | -------------- |
| 修改密码     | `POST /user/profile/change-password` | `ChangePasswordDto`   | `null`              | 任何登录用户   |
| 修改资料     | `POST /user/profile/update`      | `UpdateProfileDto`    | `UserProfile`       | 任何登录用户   |
| 审计日志列表 | `POST /user/audit-log/list`      | `AuditLogQuery`       | `AuditLogResult`    | `ADMIN` 及以上 |

### 4.4 业务约束

| 约束                       | 规则                                                     |
| -------------------------- | -------------------------------------------------------- |
| 修改密码 - 旧密码         | 必填字符串，后端校验是否匹配                             |
| 修改密码 - 新密码         | 必填字符串，最少 8 位                                    |
| 修改密码 - 响应           | 成功返回 `null`，旧密码错误返回 `INVALID_PASSWORD` 错误  |
| 修改资料 - 字段           | 三个字段均为可选，至少提供一个                           |
| 修改资料 - 昵称           | 字符串，最长 64 字符                                     |
| 修改资料 - 邮箱           | 有效邮箱格式，最长 128 字符                              |
| 修改资料 - 微信号         | 字符串，最长 64 字符                                     |
| 审计日志 - 分页           | `page` 默认 1（最大 1000），`pageSize` 默认 20（最大 100）|
| 审计日志 - 时间范围       | `startDate` / `endDate` 为 ISO 8601 格式                 |
| 审计日志 - 排序           | 按 `createdAt` 降序排列                                  |

  if (!oldPassword) { setError('请输入旧密码'); return; }

useEffect(() => {

## 十二、权限控制

| 功能               | 所需角色         | 前端守卫                                     |
| ------------------ | ---------------- | -------------------------------------------- |
| 访问个人资料页     | 任何登录用户     | 由 `AuthGuard` 统一守卫                      |
