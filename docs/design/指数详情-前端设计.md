# 指数详情（Index Detail）— 前端实现规划

> **目标读者**：AI 代码生成助手。请严格按照本文定义的组件结构、布局规则、API 调用方式实现。

---

## 一、需求背景

市场概览页已通过 `/market/index-quote` + `/market/index-trend` 展示 6 大核心指数的实时报价与走势。后端 **Index 模块** 提供了 3 个独立端点，可补全以下功能：

| 后端端点                   | 说明               | 状态      |
| -------------------------- | ------------------ | --------- |
| `POST /index/list`         | 支持的核心指数列表 | ❌ 未接入 |
| `POST /index/daily`        | 指数日线行情       | ❌ 未接入 |
| `POST /index/constituents` | 指数成分股及权重   | ❌ 未接入 |

**设计方向**：新增 **指数详情页**（`/market/index`），作为市场概览的下钻页面。用户可选择指数 → 查看日 K 线 + 成交量 → 查看成分股权重列表。同时在市场概览的指数卡片上添加跳转入口。

## 二、技术栈与约定

| 技术       | 版本/说明                                                          |
| ---------- | ------------------------------------------------------------------ |
| 框架       | React 19（Vite + SWC）                                             |
| UI 库      | MUI v7（`@mui/material`），使用 `sx` prop 做样式                   |
| 图表       | ApexCharts 4.5（`Chart` + `useChart` from `src/components/chart`） |
| API 客户端 | `apiClient.post<T>(url, body)` from `src/api/client.ts`            |
| 布局容器   | `DashboardContent` from `src/layouts/dashboard`                    |
| 路由       | 新增 `/market/index` → `IndexDetailPage`                           |

### 关键约定（与市场概览一致）

1. **API 调用**：统一 `apiClient.post<T>(url, body)`。
2. **日期参数**：`trade_date` 为 `YYYYMMDD` 格式字符串，不传则后端取最新交易日。
3. **错误处理**：组件内 try/catch + Alert 展示，不阻塞整页。
4. **加载状态**：各子组件独立管理 `loading`，用 MUI `Skeleton` 占位。
5. **图表 tooltip**：`tooltip.shared = true` 时必须同时设置 `tooltip.intersect = false`。
6. **空数据保护**：使用前检查 null / undefined / 空数组。
7. **涨跌颜色**：红涨绿跌（`error.main` 涨，`success.main` 跌）。
8. **金额单位**：Tushare 返回金额单位为千元，展示时转换为"亿元"（除以 100000）。

## 三、API 类型定义

在 `src/api/index-detail.ts` 中新增（独立文件，不并入 `market.ts`，因为属于不同 Controller）：

```typescript
import { apiClient } from './client';

// ─── /index/list ────────────────────────────────────────────

export type IndexInfo = {
  tsCode: string; // 指数代码，如 "000001.SH"
  name: string; // 指数简称
  market: string; // 交易所（SSE / SZSE / CSI 等）
  category: string; // 分类（规模 / 行业 / 主题 等）
  publishDate: string; // 发布日期
  baseDate: string; // 基期
  basePoint: number; // 基点
  listDate: string; // 上市日期
};

export function fetchIndexList() {
  return apiClient.post<IndexInfo[]>('/api/index/list', {});
}

// ─── /index/daily ───────────────────────────────────────────

export type IndexDailyQuery = {
  ts_code: string; // 必填 — 指数代码
  start_date?: string; // YYYYMMDD
  end_date?: string; // YYYYMMDD
  limit?: number; // 最大条数
};

export type IndexDailyItem = {
  tradeDate: string; // YYYYMMDD
  open: number;
  high: number;
  low: number;
  close: number;
  preClose: number;
  change: number;
  pctChg: number;
  vol: number; // 成交量（手）
  amount: number; // 成交额（千元）
};

export function fetchIndexDaily(query: IndexDailyQuery) {
  return apiClient.post<IndexDailyItem[]>('/api/index/daily', query);
}

// ─── /index/constituents ────────────────────────────────────

export type IndexConstituentQuery = {
  ts_code: string; // 必填 — 指数代码
  trade_date?: string; // YYYYMMDD，不传取最新
};

export type IndexConstituentItem = {
  tsCode: string; // 成分股代码
  name: string; // 成分股名称
  industry: string; // 所属行业
  weight: number; // 权重（%）
  close: number; // 最新收盘价
  pctChg: number; // 涨跌幅（%）
  totalMv: number; // 总市值
  circMv: number; // 流通市值
};

export type IndexConstituentResult = {
  tsCode: string;
  name: string;
  tradeDate: string;
  totalCount: number;
  constituents: IndexConstituentItem[];
};

export function fetchIndexConstituents(query: IndexConstituentQuery) {
  return apiClient.post<IndexConstituentResult>('/api/index/constituents', query);
}
```

> **注意**：以上类型为推测结构，实际接入时需根据后端真实返回字段校对。字段命名遵循项目驼峰约定（后端返回 `snake_case`，`apiClient` 拦截器统一转 `camelCase`）。

## 六、各组件详细设计

### 6.1 IndexDetailView（主视图）

```tsx
// src/sections/index-detail/view/index-detail-view.tsx

import { useState, useEffect } from 'react';

import Grid from '@mui/material/Grid';
import Stack from '@mui/material/Stack';
import Typography from '@mui/material/Typography';
import Autocomplete from '@mui/material/Autocomplete';
import TextField from '@mui/material/TextField';

import { DashboardContent } from 'src/layouts/dashboard';

import { fetchIndexList } from 'src/api/index-detail';
import type { IndexInfo } from 'src/api/index-detail';

import { IndexOverviewCard } from '../index-overview-card';
import { IndexDailyChart } from '../index-daily-chart';
import { IndexConstituentsTable } from '../index-constituents-table';
import { IndexWeightDistribution } from '../index-weight-distribution';

export function IndexDetailView() {
  // ── 指数列表 ──
  const [indexList, setIndexList] = useState<IndexInfo[]>([]);
  const [selected, setSelected] = useState<IndexInfo | null>(null);
  const [listLoading, setListLoading] = useState(true);

  useEffect(() => {
    fetchIndexList()
      .then((list) => {
        setIndexList(list ?? []);
        // 默认选中沪深300 或第一个
        const default300 = list?.find((i) => i.tsCode === '000300.SH');
        setSelected(default300 ?? list?.[0] ?? null);
      })
      .catch(() => {})
      .finally(() => setListLoading(false));
  }, []);

  const tsCode = selected?.tsCode ?? '';

  return (
    <DashboardContent>
      <Stack
        direction={{ xs: 'column', sm: 'row' }}
        alignItems={{ sm: 'center' }}
        justifyContent="space-between"
        spacing={2}
        sx={{ mb: 3 }}
      >
        <Typography variant="h4">指数详情</Typography>
        <Autocomplete
          options={indexList}
          value={selected}
          onChange={(_, v) => setSelected(v)}
          getOptionLabel={(o) => `${o.name}（${o.tsCode}）`}
          loading={listLoading}
          sx={{ width: 300 }}
          renderInput={(params) => <TextField {...params} size="small" label="选择指数" />}
        />
      </Stack>

      {tsCode && (
        <Grid container spacing={3}>
          {/* 概况卡片 — 占满一行 */}
          <Grid size={{ xs: 12 }}>
            <IndexOverviewCard tsCode={tsCode} indexInfo={selected} />
          </Grid>

          {/* 日 K 线图 */}
          <Grid size={{ xs: 12 }}>
            <IndexDailyChart tsCode={tsCode} />
          </Grid>

          {/* 成分股表格 */}
          <Grid size={{ xs: 12 }}>
            <IndexConstituentsTable tsCode={tsCode} />
          </Grid>

          {/* 权重分布 — 饼图 + 行业汇总 */}
          <Grid size={{ xs: 12 }}>
            <IndexWeightDistribution tsCode={tsCode} />
          </Grid>
        </Grid>
      )}
    </DashboardContent>
  );
}
```

### 6.3 IndexDailyChart（日 K 线图）

```
文件：src/sections/index-detail/index-daily-chart.tsx
Props：{ tsCode: string }
```

**数据来源**：`fetchIndexDaily({ ts_code: tsCode, limit: N })` from `src/api/index-detail.ts`。

**交互**：

- **周期切换**：`ToggleButtonGroup`，选项 `60 | 120 | 250 | 500`（日），对应约 3月/半年/1年/2年。默认 `250`。
- 切换周期时重新请求 `limit` 参数。

**图表**：ApexCharts **混合图表**（`type="line"` 容器）

```tsx
const series = [
  {
    name: '收盘价',
    type: 'area',
    data: dailyData.map((d) => d.close),
  },
  {
    name: '成交量（亿）',
    type: 'column',
    data: dailyData.map((d) => d.amount / 100000), // 千元 → 亿元
  },
];
```

- **Y 轴**：双 Y 轴。左侧 = 收盘价（`yaxis[0]`），右侧 = 成交量（`yaxis[1]`，`opposite: true`）。
- **X 轴**：`categories = dailyData.map(d => fmtDate(d.tradeDate))`，`tickAmount: 10`。
- **Area 样式**：`fill.type = 'gradient'`，蓝色主题色。
- **Column 样式**：column 宽度 `'50%'`，半透明灰色 `varAlpha(theme.vars.palette.grey['500Channel'], 0.3)`。
- **Tooltip**：`shared: true, intersect: false`。

加载状态：`Skeleton variant="rectangular" height={360}`。

### 6.4 IndexConstituentsTable（成分股权重表）

```
文件：src/sections/index-detail/index-constituents-table.tsx
Props：{ tsCode: string }
```

**数据来源**：`fetchIndexConstituents({ ts_code: tsCode })` from `src/api/index-detail.ts`。

**交互**：

- **排序**：点击表头排序（按权重降序为默认）。支持排序列：`weight`、`pctChg`、`totalMv`。

**表格列**：

| 列        | 字段       | 对齐   | 格式                |
| --------- | ---------- | ------ | ------------------- |
| 排名      | 前端计算   | center | 序号                |
| 代码      | `tsCode`   | left   | 原始值              |
| 名称      | `name`     | left   | 原始值              |
| 行业      | `industry` | left   | `Label` 色标        |
| 权重（%） | `weight`   | right  | 两位小数            |
| 收盘价    | `close`    | right  | 两位小数            |
| 涨跌幅    | `pctChg`   | right  | `fPctChg`，红涨绿跌 |
| 总市值    | `totalMv`  | right  | `fQianYuan`（亿元） |
| 流通市值  | `circMv`   | right  | `fQianYuan`（亿元） |

- 使用 `Scrollbar` + `TableContainer` 包裹，`minWidth: 800`。
- 显示表头统计：`共 {totalCount} 只成分股 · 数据日期 {tradeDate}`。

加载状态：表格 `Skeleton`。

### 6.5 IndexWeightDistribution（权重分布可视化）

```
文件：src/sections/index-detail/index-weight-distribution.tsx
Props：{ tsCode: string }
```

**数据来源**：复用 `IndexConstituentsTable` 的数据（通过父组件状态提升传递，或组件内独立请求均可；推荐由父组件统一请求后 props 传入以避免重复请求）。

> **实现建议**：为减少重复请求，`IndexDetailView` 中将 `constituents` 数据状态提升，同时传给 `IndexConstituentsTable` 和 `IndexWeightDistribution`。或者此组件直接接收 `constituents: IndexConstituentItem[]` 作为 props。

**布局**（`Grid` 一行两列）：

| 左半区域（md=5）        | 右半区域（md=7）                 |
| ----------------------- | -------------------------------- |
| **Top 10 权重饼图**     | **行业权重汇总表**               |
| ApexCharts `type="pie"` | 小型 Table：行业、总权重、成分数 |
| labels = Top 10 name    | 按权重降序排列                   |
| series = Top 10 weight  | 行业名称 + 水平 Bar 显示占比     |

**饼图**：

- `labels`: 取权重 Top 10 成分股 `name`，其余合并为 "其他"。
- `series`: 对应 `weight` 值。
- `colors`: 使用主题调色板（`theme.palette.primary.main` 渐变）。
- `legend`: `position: 'bottom'`, `horizontalAlign: 'center'`。

**行业汇总表**：

- 前端从 `constituents` 按 `industry` 分组，累加 `weight`，计算每个行业的成分股数量。
- 列：`行业名称 | 权重合计（%） | 成分股数 | 权重条（内联 LinearProgress）`。
- 排序：按权重合计降序。

## 九、注意事项

1. **API 字段校对**：第三节的类型定义为推测结构，实际实现前需与后端确认返回字段。特别是 `/index/constituents` 的字段可能与上述不完全一致。
2. **性能**：沪深 300 有 300 只成分股、中证 1000 有 1000 只。成分股表格需要：
   - 使用前端过滤/搜索，不要每次搜索都请求后端。
   - 考虑虚拟滚动（若成分股超过 500 只）或分页展示（50 条/页）。
3. **饼图限制**：Top 10 + "其他"，不要展示所有成分股在饼图中。
4. **复用现有工具函数**：`fPctChg`、`fQianYuan`、`fNumber` 等来自 `src/utils/format-number.ts`。
5. **图标**：使用 `solar:chart-bold` 作为导航图标。如果需要新增图标，按照 `src/components/iconify/` 的注册流程添加。
6. **URL 参数**：从市场概览跳转时携带 `?code=xxx`，指数详情页读取并设为默认选中。使用 `useSearchParams` (react-router) 或 `window.location.search` 解析。
